# F-25 音频转文字（ASR）设计

> 日期：2026-07-22  
> 状态：✅ 设计已与用户确认，待从 0 带学实现  
> 关联：后续再做文字转语音（TTS）；视频抽音轨延后  
> 教学约定：按层拆分、带学带写，不一次性交完代码

---

## 1. 已确认的产品决策

| 决策点 | 结论 |
|--------|------|
| 学习 / 实现顺序 | **先 ASR，后 TTS** |
| 第一版范围 | **仅音频文件**（mp3 / wav 等） |
| 视频抽音轨 | **不做**；等做视频能力时再上 |
| 转写结果 | 要 **带时间轴** 的句子，不只全文 |
| 点击句子 | **可以**：跳到该句 `startMs` 并播放 |
| 悬停句子 | **要**：鼠标悬停时该句高亮 |
| 播放中自动跟读高亮 | **第一版不做**（可后续加） |
| 音频摘要 / 章节摘要 | **本阶段不做**（Phase B） |
| 入口 | 复用现有「建立索引」，不另起一套上传转写流程 |

### 关于「抽音轨」（备忘，本阶段不做）

- 纯音频（如 mp3）：已经是声音，无需抽轨  
- 视频（如 mp4）：画面 + 声音绑在一起，需用 ffmpeg 抽出音轨再送 ASR  
- 本阶段明确延后，避免干扰主干学习

---

## 2. 目标（MVP 验收）

用户侧：

1. 网盘中的 **音频文件** 可触发「建立索引」
2. 索引完成后可看到 **转写文稿（按句）**
3. 打开音频预览：文稿列表  
   - **悬停** → 该行高亮  
   - **点击** → `audio.currentTime = startMs / 1000` 并播放  
4. 普通文档（PDF/Word 等）索引行为 **无回归**

技术侧：

- chunk 含非空 `content` 与合理的 `startMs` / `endMs`
- 提供读文稿接口（见下文 API）
- 长音频可分段转写后再拼时间轴

---

## 3. 架构

```
上传音频到网盘
  → 用户点「建立索引」
  → 进入现有 document-index 队列（BullMQ + Worker）
  → Worker 按 mime / 类型判断走「音频分支」：
       1. Storage 读出本地文件
       2. （可选但推荐）ffmpeg → 16kHz 单声道 wav；过长则按时长切段
       3. ASR Provider 调云 API
            → { text, segments: [{ text, startMs, endMs }] }
       4. 按句（或合并过短句）写入 DocumentChunk（带时间戳）
       5. embedding（与文档路径对齐，便于日后检索）
  → 状态 ready
  → 前端播放器旁展示 transcript；悬停高亮、点击跳播
```

**核心思想：**  
音频当作「另一种可索引文件」。尽量复用现有索引流水线；只在「抽文本」这一步换成 ASR，而不是另起一套 Job。

---

## 4. 数据模型

复用表 `DocumentChunk`，增加两个 **可选** 字段：

| 字段 | 类型 | 含义 |
|------|------|------|
| `content` | 已有 | 这一句 / 段的转写文字 |
| `chunkIndex` | 已有 | 排序 |
| `startMs` | `Int?` | 在音频中的起始时间（毫秒） |
| `endMs` | `Int?` | 结束时间（毫秒） |

说明：

- 普通 PDF/Word 的 chunk：`startMs` / `endMs` 为 `null`，不影响现有功能  
- 音频：每条（或合并后的）句子一条 chunk，带时间戳  
- 过短句子可合并到 ≥N 字，避免列表过碎（实现时定阈值）

Prisma migration：新增列即可（如 `start_ms` / `end_ms`）。

---

## 5. API（读文稿）

建议专用接口（比拼 extracted-text 更清晰）：

```
GET /api/files/:id/ai/transcript
→ {
    segments: [
      { text: string, startMs: number, endMs: number }
    ]
  }
```

前端用这份列表渲染；点击时用对应 `startMs`。

（实现时也可先复用已有 chunk 查询再组装；对外形态以 transcript 为准。）

## 6. 实现分层（带学顺序）

### 6.1 ASR Provider

- 目录建议：`src/files/ai/asr/`
- 职责：本地音频 → 云 ASR → 统一结构  
  `{ text, segments: [{ text, startMs, endMs }] }`
- 上游：硅基流动 Whisper 兼容接口（如 SenseVoice）
- 配置：`AI_ASR_BASE_URL` / `AI_ASR_MODEL` / `AI_ASR_API_KEY`  
  Key 可回退：Vision → Embedding → `AI_API_KEY`
- 必须带单测（mock HTTP），方便不连真网也能学

### 6.2 媒体预处理（ffmpeg）

- 稳妥路径：转成 **16kHz 单声道 wav** 再送 ASR  
- 复用项目已有 ffmpeg  
- 长音频：切段 → 逐段转写 → 按段起始偏移校正 `startMs`/`endMs`  
- 本阶段 **不做视频抽音轨**

### 6.3 接到现有索引流水线

- 允许音频 mime 进入索引  
- extractor / processor：音频走 ASR，不走 PDF/Office 解析  
- 写 chunk 时写入 `startMs` / `endMs`  
- embedding 步骤与文档对齐（便于以后「搜这句话」）

### 6.4 前端（音频预览）

- 拉取 `transcript`
- 列表展示每句（可显示简易时间码）
- **悬停**：CSS 高亮该行  
- **点击**：跳转 `currentTime` 并 `play()`  
- **不做（MVP）**：根据播放进度自动高亮当前句

---

## 7. 明确不做（本阶段）

- 视频抽音轨  
- 音频 / 章节摘要  
- TTS、划词朗读（另案，ASR 完成后再做）  
- 播放中自动跟读高亮  

---

## 8. 建议实现顺序（教学用）

1. Prisma：`startMs` / `endMs` + migration  
2. ASR Provider + 单测（mock）  
3. ffmpeg 预处理工具 + 单测  
4. 索引 Worker 音频分支（真正跑通一条 mp3）  
5. `GET .../ai/transcript`  
6. 前端：文稿列表 + 悬停高亮 + 点击跳播  

每步建议：**先写测 → 再写最小实现 → 你本地跑通 → 再进下一步。**
