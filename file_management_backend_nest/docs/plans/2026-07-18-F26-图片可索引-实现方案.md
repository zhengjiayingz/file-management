# F-26 图片可索引 — 实现方案（预览版）

> 日期：2026-07-18  
> 用途：完整预览实现对照；用户手动回滚后可按本方案从 0 带写。  
> PRD： [2026-07-08-ai-features-prd.md](./2026-07-08-ai-features-prd.md) § F-26（**v1.17 含 OCR 划词面板增补**）

## 目标

png / jpg / jpeg / webp / gif → OCR（硅基 `DeepSeek-OCR`）→ 复用现有 chunk / embed / 摘要 / RAG 流水线；图片预览挂 `DocumentAiPanel`。

**增补：** 索引 `ready` 后，图片预览增加 **OCR 文本中间栏**，在此划词并复用右侧划词问答；**不改变** Word / TXT / PDF 的左右两栏布局。

## 后端

| 步骤 | 文件 | 要点 |
|------|------|------|
| 1 | `src/files/ai/vision/vision.provider.ts` | `extractTextFromImage(buffer, mime)` → `POST {AI_VISION_BASE_URL}/chat/completions`，`image_url` + `Free OCR.` |
| 2 | `vision.provider.spec.ts` | mock `fetch`：成功 / HTTP 失败 / 空文本 |
| 3 | `chunk/text-extractor.ts` | `isIndexableTextDocument` 增加图片扩展名+MIME；`extractTextFromBufferInternal` 图片分支调 OCR |
| 4 | `files-ai-index.service.ts` | 400 文案含「可 OCR 的图片」 |
| 5 | e2e `files-ai-index` | png → pending；mp4 → 400 |
| 6 | `GET .../ai/extracted-text` | chunks 按 `joinOverlappingChunkContents` 拼近似全文，供 OCR 划词面板 |

## 环境变量（已有 `.env.example`）

```
AI_VISION_BASE_URL=https://api.siliconflow.cn/v1
AI_VISION_API_KEY=...   # 可与 Embedding 同一把硅基 Key
AI_VISION_MODEL=deepseek-ai/DeepSeek-OCR
```

## 前端

| 步骤 | 文件 | 要点 |

| 3 | i18n | `preview.imageTitle`、`preview.ocrPanel*` |
| 4 | **不改** | `PdfDocumentPreviewDialog` / `TextChunkPreviewDialog` / `OfficePreviewDialog` 仍左右两栏 |

## 预览验收

1. 上传带文字的 png/jpg → 双击打开预览 → 右侧「建立索引」  
2. Worker 跑通后 status=`ready`，中间栏出现 OCR 文本；划词 + 右侧划词问答可用；RAG 能命中 OCR 关键词  
3. 空白图或 OCR 失败 → `failed` + 友好错误  
4. 打开 Word/TXT/PDF：仍为左右两栏，原有划词正常  

## 不做（本特性）

扫描 PDF（F-30）、截图解题 VL（F-27）、bmp 索引、图片轮播多图切换、**图片像素框选 / OCR 坐标映射**。
