**Goal：** 先让网盘文件预览中的划词 / RAG / 解题对话可落盘续聊；再提供临时截图通道；最后用浏览器插件完成「截图问答→存网盘」与「截图解题→存错题本」。

**Architecture：** 单文件 AI 会话挂在 `userFileId + mode`；临时截图用 `tempImageId`（TTL）承载同形态会话；保存时分支转正——问答导出为网盘文件包，解题走 `wrong-questions/from-temp`。插件只做截图与侧栏 UI，业务 API 在 Nest。

**Tech Stack：** NestJS、Prisma、现有 VL/OCR/划词/RAG 流式、Chrome/Edge Extension（MV3）、JWT、Vue3 预览面板。

## 3. 阶段 A — 预览对话落盘（地基）

### 3.1 数据模型

```prisma
enum FileAiChatMode {
  selection
  rag
  solve
}

enum FileAiChatRole {
  user
  assistant
}

model FileAiChatSession {
  id         Int            @id @default(autoincrement())
  userId     Int            @map("user_id")
  userFileId Int            @map("user_file_id")
  mode       FileAiChatMode
  createdAt  DateTime       @default(now()) @map("created_at")
  updatedAt  DateTime       @updatedAt @map("updated_at")

  user     User               @relation(...)
  userFile UserFile           @relation(...)
  messages FileAiChatMessage[]

  @@unique([userId, userFileId, mode])
  @@index([userFileId, mode])
  @@map("file_ai_chat_sessions")
}

model FileAiChatMessage {
  id        Int            @id @default(autoincrement())
  sessionId Int            @map("session_id")
  role      FileAiChatRole
  content   String         @db.Text
  /// selection 模式可选：当时划词片段；rag 可选存 citations JSON
  meta      Json?
  createdAt DateTime       @default(now()) @map("created_at")

  session FileAiChatSession @relation(..., onDelete: Cascade)

  @@index([sessionId, createdAt])
  @@map("file_ai_chat_messages")
}
```

约束：

- 同一 `(userId, userFileId, mode)` **仅一条** session（upsert / 懒创建）。
- 流式 **完成后** 再写 assistant 全文；失败或 abort **不**写半截。
- 删网盘文件：级联删 session（`onDelete: Cascade`）或先禁预览再清理——实现选 Cascade。
- 消息上限（建议）：每会话保留最近 **100** 条；超出从最早 trim。发给模型仍 `takeLastMessages(N=6)`（解题）或现有划词/RAG N。

### 3.2 API（阶段 A）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/files/:id/ai/chat-sessions/:mode` | 懒创建不存在则空消息；返回 `{ sessionId, mode, messages[] }` |
| DELETE | `/api/files/:id/ai/chat-sessions/:mode` | 清空该模式会话（删消息或重建） |
| POST | `/api/files/:id/ai/chat-sessions/:mode/messages` | body: `{ role, content, meta? }`；用于流式结束后落库（或服务端在 ask/solve 结束钩子内写，二选一） |

**推荐落库钩子（更省前端差错）：**

- 现有 `ask-selection` / `rag-ask` / `solve-math` 在流式成功结束后，由服务端追加 user+assistant（需请求带完整 user 文案；assistant 用拼接缓冲）。
- 若保持纯前端落库：流式 `onDone` 再 `POST messages` 两次；须防重复提交。

打开预览：`DocumentAiPanel` 按当前 `chatMode` **GET** 历史再渲染；关 Dialog **不必**再 POST。

### 3.3 前端改动要点

| 文件 | 改动 |
|------|------|
| `src/api/ai.ts` | 增会话 GET/DELETE/POST messages |
| `DocumentAiPanel/index.vue` | mount / `watch(chatMode)` / `watch(fileId)` 拉历史；流式成功确认落库；提供「清空本模式对话」 |
| 预览 Dialogs | 无需改销毁策略；历史已在服务端 |

### 3.4（A 可选增强）预览「导出问答到网盘」

在已有 `userFileId` 上：

1. 取当前模式 session 消息 → 生成 Markdown（含时间、模式、原文）。
2. `POST` 上传新文件（如 `{原名}-问答-{date}.md`）到同目录或用户选定目录。
3. **不**复制原图（图已在网盘）；插件路径才需要「图+文」一起转正。

此按钮可放阶段 A 末或 B 与插件一并做；方案预留 UI 文案 `preview.exportChatToDrive`。

## 4. 阶段 B — tempImageId 与两条保存路径

> API 名与路径 **锁定对齐** F-29 方案 §10；本方案补充「问答会话」与「导出网盘」。

### 4.1 临时对象与会话

- 表 `MathTempImage`（推荐）：`tempImageId`、`userId`、`storageKey`、`mime`、`expiresAt`、`consumedAt?`
- **Temp 会话：** 复用 `FileAiChatSession` 形状困难（无 `userFileId`）→ 二选一：
  - **B1（推荐）：** `TempAiChatSession` / `TempAiChatMessage`（`tempImageId` + `mode`），字段与 File 版对称。
  - **B2：** `FileAiChatSession.userFileId` 改为可空，另加 `tempImageId?`，check 约束二选一 —— 单表但易脏。

**推荐 B1**，转正时可选「把 temp 消息复制到新 file 的 FileAiChatSession」。

### 4.2 API（在 §10 基础上扩展）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/ai/math-temp-images` | multipart → `{ tempImageId, expiresAt }` |
| POST | `/api/ai/math-temp-images/:tempImageId/solve` | 解题流（mode=solve） |
| POST | `/api/ai/math-temp-images/:tempImageId/ask` | **新增**：自然语言问答流（mode=selection 语义，无划词则可整图问） |
| GET | `/api/ai/math-temp-images/:tempImageId/chat-sessions/:mode` | temp 会话历史 |
| DELETE | `/api/ai/math-temp-images/:tempImageId` | 丢弃 |
| POST | `/api/wrong-questions/from-temp` | **路径 ②** 存错题本（已钉） |
| POST | `/api/ai/math-temp-images/:tempImageId/save-to-drive` | **路径 ①** 存网盘（本方案钉名） |

### 4.3 路径 ①：截图问答 → 存网盘

```text
tempImageId + mode=selection（或 ask）会话
  → POST .../save-to-drive
       { folderId?, fileName?, includeTranscript: true }
  → 事务：
       1. 临时图转正 → UserFile（进入网盘列表）
       2. 写入问答 Markdown（同目录或 sibling）
       3. 可选：复制 Temp 消息 → FileAiChatSession(selection)
       4. 标记 temp consumed / 删除临时对象
  → 返回 { userFileId, transcriptFileId? }
```

默认文件名建议：`截图问答-YYYYMMDD-HHmm.png` + `.md`。

### 4.4 路径 ②：截图解题 → 存错题本

```text
tempImageId + mode=solve 会话（取最后 assistant 为 answerText）
  → POST /wrong-questions/from-temp
       { tempImageId, answerText, tags?, difficulty?, questionText? }
  → 事务：转正 UserFile + WrongQuestionBookItem + consume temp
  → 题干：可 OCR；与 F-29 网盘存入一致
```

**不**自动把全部追问写入 `answerText`（与 F-29 一致）；用户要更新解答可另 PATCH。

### 4.5 鉴权 / CORS

- Bearer JWT（扩展 storage 或登录页拿 token）。
- Nest CORS 增加扩展 origin / `chrome-extension://*` 策略在阶段 C 定；B 阶段可用同源 Demo 页验证。
