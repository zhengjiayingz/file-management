**Goal：** 在网盘上交付可面试演示的三条能力——（1）扫描 PDF 可索引并复用全文问答/摘要；（2）以图搜图 + 以文搜图；（3）跨类型近似查重（分组列出、用户多选后手动删除）。指纹/视觉向量与 RAG「建立索引」分轨，避免演变成上传即全量 AI 索引。

**Architecture：**  
- **重能力（按需）：** 现有 `DocumentIndexJob` —— 用户点「建立索引」→ 抽字/OCR → chunk/embed → RAG/摘要。F-30 仅补扫描 PDF 渲页 OCR 接入该流水线。  
- **轻能力（入库可自动）：** 文件指纹 Job —— content hash、图片 pHash、（可选）短文本/图像 CLIP 向量，供查重与以图/以文搜图。  
- **检索：** 文档语义搜复用 F-08；图片检索走 `FileFingerprint.imageEmbedding` + 内存余弦 Top-K（MVP，对齐 F-08 规模假设；F-38 已落地）。  
- **查重：** 独立页按「重复组」展示全组成员，多选后调现有删除/回收站，不自动删。

**Tech Stack：** NestJS、Prisma/MySQL JSON 向量、BullMQ（指纹 Job 待补 / 索引 Job）、现有 OCR（F-26）、硅基 VL Embedding（F-38 已用）、Vue3 查重页待做 + 搜索 Dialog 已扩展以图搜、JWT。

**工期粗估（单人）：** 完整 MVP **约 3～4 周（14～18 人天）**。

## 1. 已确认的产品决策

### 1.1 能力包（本期要做）

| # | 能力 | 说明 |
|---|------|------|
| **①** | **F-30 扫描 PDF 可索引** | 无文字层 PDF：逐页渲染 → OCR → 拼全文 → 现有 index；之后 **RAG / 摘要** 复用现有，不新做产品 |
| **②** | **以图搜图** | ✅ 上传查询图 → Top-K 相似**图片**（MVP：搜索时按需指纹；候选≤100） |
| **③** | **以文搜图** | ❌ **砍掉**（2026-07-21：跨模态效果差） |
| **④** | **文档语义搜索** | **复用 F-08**，不重做；统一搜索 UI 可分区展示「文档 + 图片」 |
| **⑤** | **近似查重** | 文本类 + 图片；分组列出全部候选；用户勾选后手动删除 |

### 1.2 索引 vs 指纹（关键，防范围膨胀）

| 轨道 | 触发 | 产出 | 用于 |
|------|------|------|------|
| **RAG 索引**（重） | 用户显式「建立索引」 | chunks + 文本 embedding + 可摘要 | 全文问答、摘要、知识卡、F-08 文档搜 |
| **文件指纹**（轻） | **新上传默认参与**；查重页可「扫描全部」补算 | content hash、图片 pHash、图像向量、（可选）短文本指纹向量 | 查重、以图/以文搜图 |

**明确否决：** 查重/搜图 **不要求** 网盘每个文件都先点「建立索引」。  
**明确否决：** 本期 **不** 默认「上传即自动跑完整 RAG 索引」。若将来要对某类文件自动索引，另立决策。

「扫描全部」= 对文件补算 **指纹**，不是给全部文件建 RAG。

### 1.3 历史数据与查重范围

- **默认：** 仅 **本功能上线后新上传**（或指纹 Job 启用后新写入）的文件自动算指纹并参与查重/搜图库。  
- **可选：** 查重页提供 **「扫描全部」** —— 手动触发历史文件补算指纹（Demo 用）；默认不自动全盘。  
- 无指纹的文件：不出现在近重组 / 图片检索候选中（可在 UI 提示「未建指纹」）。

### 1.4 查重 UX（已锁定）

- 入口：**独立页**（侧栏「重复文件」或同等一级入口）；**不以设置页为主入口**。  
- 近似去重：超阈值进同一 **重复组**。  
- 一组列出 **全部** 成员（例：3 个重复则列 3 个）。  
- **不自动删除**；用户多选（如勾 2 个）→「删除」→ 走现有删除/回收站。  
- 建议校验：禁止一组内删光全部（至少保留 1 个）或强确认。  
- 上传弱提醒（可选增强）：命中近似时 Toast「发现相似文件，去查看」→ 跳到该组；**非唯一入口**。

### 1.5 近重策略（实现约定）

| 类型 | 精确 | 近似 |
|------|------|------|
| **通用文件** | 内容 hash（相同则一组，标「完全重复」） | — |
| **图片** | pHash 距离 = 0 或极低 | pHash 近距 **和/或** 图像向量余弦 ≥ 阈值 |
| **文本 / 已抽字文档** | 文本规范化后 hash | 短文本或全文指纹 embedding 余弦 ≥ 阈值（**不必**依赖 RAG chunk 表；可与指纹任务一并写入） |
| **扫描 PDF** | 无 OCR 前仅能文件 hash；有指纹 OCR 抽字后可进文本近重 | 查重扫描时可 **只 OCR+指纹**，仍可不建 RAG |

阈值进 env（如 `DEDUP_IMAGE_COSINE_MIN`、`DEDUP_TEXT_COSINE_MIN`、`DEDUP_PHASH_MAX_DISTANCE`）。

### 1.6 搜图范围（MVP）

- 候选：**网盘图片**（png/jpg/webp/gif 等，与 F-26 可索引图对齐）。  
- **本期不做：** PDF 页级以图搜、视频帧、区域框选搜图、独立向量数据库。  
- 规模：千级以内 MySQL JSON + 内存 Top-K（对齐 F-08）。

### 1.7 非目标（本期）

- ❌ 上传自动完整 RAG 索引  
- ❌ 查重自动删除 / 静默合并  
- ❌ 浏览器插件（已押后）  
- ❌ pgvector / Qdrant（量大再迁）  
- ❌ 以文搜图与 F-08 混成单一向量空间（文本文档仍用文本 embedding；图片用 CLIP 空间）  
- ❌ **AI 整理文件夹 / 网盘 Agent**（PRD **F-40 / F-20**，远期不排期）

## 3. 架构要点

### 3.1 数据（建议）

```text
FileFingerprint（或拆表）
  userFileId, userId
  contentHash?
  pHash?                 // 仅图片
  imageEmbedding Json?   // CLIP 图像向量
  textFingerprint Json?  // 可选：短文本/抽字指纹向量（非 RAG chunk）
  fingerprintStatus      // pending | ready | failed
  createdAt, updatedAt

查重接口不持久化「组」亦可：扫描时内存聚类返回 groups[]
```

图像检索只读 `imageEmbedding` 且 `ready` 的本人图片。

### 3.2 API 草图（落地时可微调路径）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/ai/image-search/by-image` | multipart 查询图 → Top-K 图片 |
| ~~POST~~ | ~~`/api/ai/image-search/by-text`~~ | ❌ 已砍掉 |
| GET/POST | `/api/files/duplicates/scan` | `scope=new|all`；返回 `groups[]` |
| （复用） | 现有删除接口 | 前端多选后逐个或批量删 |
| （复用） | `POST /files/:id/ai/index` | F-30 后扫描 PDF 可成功 |
| （复用） | F-08 `GET /files/search` | 文档语义搜 |

`groups[]` 形如：

```json
{
  "groups": [
    {
      "groupId": "…",
      "kind": "exact_hash" | "near_image" | "near_text",
      "items": [
        { "userFileId": 1, "fileName": "a.png", "score": 1.0 },
        { "userFileId": 2, "fileName": "b.png", "score": 0.93 },
        { "userFileId": 3, "fileName": "c.png", "score": 0.91 }
      ]
    }
  ]
}
```

### 3.3 前端

| 页面/入口 | 行为 |
|-----------|------|
| 侧栏 **重复文件** | 触发扫描、分组列表、组内多选、删除；范围切换「仅新文件指纹 / 扫描全部补指纹」 |
| 智能搜索（扩展现有 Dialog） | 查询为文：文档命中（F-08）+ 图片命中（以文搜图）；可提供「以图搜」上传控件 |
| 图片预览（可选） | 「找相似图片」→ 以图搜图 |
| PDF 预览 | 扫描件建立索引成功后，摘要/RAG Tab 与文字 PDF 一致；无文字层时划词可提示不支持 |

### 3.4 F-30（扫描 PDF）

- `extractPdfText` 文字过少时 fallback：`pdf` 逐页渲图 → `extractTextFromImage` → 拼接（`--- Page n ---`）。  
- env：`AI_MAX_OCR_PDF_PAGES`（默认如 50）；超限明确失败文案。  
- 文字层足够则 **不** OCR。  
- 进度文案可选：`OCR 第 n/m 页`。  
- 验收：mock/真扫描 2 页 → index ready → rag-ask / summary 可用。
