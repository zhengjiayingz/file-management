# F-29 错题本 — 实现方案

> 日期：2026-07-19  
> 状态：✅ 预览实现已落地并经手测增强（难度 / 筛选 / 刷题·题解 / 多考点）  
> 关联：[2026-07-08-ai-features-prd.md](./2026-07-08-ai-features-prd.md) § F-29  
> 前置：[2026-07-18-F27-截图数学解题-产品约定.md](./2026-07-18-F27-截图数学解题-产品约定.md)（临时上传中间态记入后续）  
> 前置实现：[2026-07-18-F27-截图数学解题-实现方案.md](./2026-07-18-F27-截图数学解题-实现方案.md)  

**Goal：** 网盘图片解题后可将题干/解答/难度/考点标签持久化为错题；独立列表页按考点·难度·时间筛选；详情支持「刷题 / 题解」模式与二次流式追问（仍带原图）。

**Architecture：** 独立 Nest 模块 `wrong-questions` + Prisma 表；存入绑定 `userFileId`；追问复用 `streamMathVisionChat`（与 F-27 同 VL 链路）；**不进** index Worker。前端：解题面板「存入错题本」+ `/wrong-questions` 列表/详情。

**Tech Stack：** NestJS、Prisma、现有 `math-vision.provider`、JWT 鉴权、Vue3、`streamSolveMath`/`readTextStream` 模式、KaTeX/`renderMarkdown`。

## 1. 已确认的决策

### 1.1 UX 方案（用户选定）

- **选定：方案 A** — 解题面板「存入错题本」+ 独立错题本列表页（浏览 / 详情 / 二次追问）；原题图绑网盘 `fileId`。
- **用户主路径：** 打开网盘图 → 解题/追问 → 「存入错题本」（可选难度 + 多考点）→ 侧栏进错题本 → 列表筛选查询 → 详情「刷题」先看题 /「题解」看 AI 答案 → 可再追问 / 改考点与难度 / 删除。
- **未选方案（归档）：**
  - B — 无独立列表、仅网盘标签筛选：复习入口弱，难做考点标签与二次追问会话。
  - C — 本迭代含临时截图 + `tempImageId`：范围过大，与本期「先闭环网盘」冲突。

### 1.2 技术方案（用户选定）

- **选定：路线 Y1** — 独立 `wrong-questions` 模块 + Prisma 表 + CRUD；`userFileId` 业务上 MVP 必填；二次追问复用 `streamMathVisionChat`；`tempImageId` **本迭代不实现逻辑**，接口形状见 **§10 F-29+**（实现时表字段按 §4.3 / §10.3 预留，勿堵路径）。
- **关键技术点：** Prisma migration、Nest 功能域模块、流式 follow-up、侧栏路由、面板存入、i18n；增强见 **§1.4**。
- **未选路线（归档）：**
  - Y2 — 本迭代做完临时中间态：工期与存储策略超出 MVP（改为 F-29+ / 插件前置）。
  - Y3 — 无独立表、用 UserFile 标签冒充：考点/解答持久化与列表体验差。

### 1.3 与 PRD 对齐说明

PRD 写「临时解题图（后续）」——与本期 Y1 一致：**不实现** temp 转正；验收以「网盘图解题 → 保存 → 列表 → 追问」为准。浏览器插件截图解题依赖 §10，排在 F-29 之后单独立项。

### 1.4 验收中追加的产品约定（2026-07-19 手测增补）

以下为预览落地后用户明确要求、已并入实现的增强（相对初版 MVP）：

| 项 | 约定 |
|----|------|
| **难度** | 枚举：`easy` / `medium` / `hard`（简单 / 中档 / 困难）；默认 `medium`。存入时可选；详情可改（变更即 PATCH）；列表可按难度筛选 |
| **考点（多选）** | 一道题可挂多个考点；设置 UI 为**可输入下拉**（`el-select` multiple + filterable + allow-create）：可选已有考点，也可输入新考点回车添加。存入面板与详情编辑一致。选项来源：本人错题本已用过的 tags |
| **列表筛选** | 支持考点、难度、**存入时间**（`createdFrom` / `createdTo`，含首尾日）。筛选控件改动**不立刻请求**；须点「查询」才带条件拉列表。「重置」清空表单并立刻无条件再查一遍 |
| **详情模式** | **刷题**：隐藏 AI 参考题解，提示可切换；**题解**：展示 `answerText`（KaTeX）。默认进入刷题模式。二次追问区两种模式均保留 |

## 4. 技术设计

### 4.1 架构与数据流

```text
[DocumentAiPanel solve] --POST /wrong-questions--> Prisma WrongQuestionBookItem
                                                      │ userFileId → UserFile → Storage
[WrongQuestion 详情追问] --POST /wrong-questions/:id/follow-up-->
                                                      │ 读图 → streamMathVisionChat → text stream
```

### 4.2 API

| 方法 | 路径 | 请求 | 响应 | 鉴权 |
|------|------|------|------|------|
| POST | `/api/wrong-questions` | `{ userFileId, questionText?, answerText, tags?: string[], difficulty?: 'easy'\|'medium'\|'hard' }` | 创建后的 item | JWT |
| GET | `/api/wrong-questions` | query: `page?` `pageSize?` `tag?` `difficulty?` `createdFrom?` `createdTo?` | `{ items, total, page, pageSize }` | JWT |
| GET | `/api/wrong-questions/:id` | — | item（含只读 `fileName` / `imageAvailable`） | JWT |
| PATCH | `/api/wrong-questions/:id` | `{ questionText?, answerText?, tags?, difficulty? }` | 更新后 item | JWT |
| DELETE | `/api/wrong-questions/:id` | — | 204 | JWT |
| POST | `/api/wrong-questions/:id/follow-up` | `{ question, messages?: {role,content}[] }` | **纯文本流**（同 solve-math） | JWT |
| POST | `/api/wrong-questions/:id/refresh-question` | — | OCR 后 item | JWT |

说明：

- `messages` 截断策略对齐 F-27：`takeLastMessages` N=6。
- follow-up **必须**每次带原题图（从 `userFileId` 读 storage → data URL）。
- 列表/详情过滤 `userId === currentUser`；跨用户 404。
- `createdTo` 按**当日结束** `23:59:59.999` 闭区间（对齐网盘/日志筛选习惯）。
- `tags`：服务端 `normalizeTags`（trim、去空、去重，最多 20）。

### 4.3 数据 / 存储

Prisma（示意）：

```prisma
enum WrongQuestionDifficulty {
  easy
  medium
  hard
}

model WrongQuestionBookItem {
  id           Int                      @id @default(autoincrement())
  userId       Int                      @map("user_id")
  // F-29 MVP：创建 校验要求必填；F-29+：与 temp 转正后的网盘图关联
  userFileId   Int?                     @map("user_file_id")
  questionText String                   @map("question_text") @db.Text
  answerText   String                   @map("answer_text") @db.Text
  tags         Json                     @default("[]") // string[]
  difficulty   WrongQuestionDifficulty  @default(medium)
  createdAt    DateTime                 @default(now()) @map("created_at")
  updatedAt    DateTime                 @updatedAt @map("updated_at")

  user     User      @relation(...)
  userFile UserFile? @relation(...)

  @@index([userId, createdAt])
  @@map("wrong_question_book_items")
}
```

- **F-29 实现约定：** DB 列按 `Int?` 建好；`POST /wrong-questions` **业务校验**仍要求 `userFileId`（无则 400）。避免 F-29+ 再改非空约束。
- migration：`20260719150000_add_wrong_question_book_items`；`20260719172000_add_wrong_question_difficulty`。
- 删除错题本条目：**不**级联删网盘文件。
- 若用户删网盘图：错题条目保留文本；`userFile` 缺失时详情降级（追问不可用）。

### 4.4 前端

| 区域 | 路径 / 说明 |
|------|-------------|
| API | `src/api/wrong-questions.ts` |
| 面板 | `DocumentAiPanel` solve：难度下拉 + 考点多选可输入下拉 +「存入」；`answerText` = 最后一条完整 assistant；题干由服务端 OCR |
| 列表 | `views/wrong-questions/index.vue`：草稿条件 +「查询」「重置」；空态区分无数据 / 过滤无匹配 |
| 详情 | `views/wrong-questions/detail.vue`：刷题/题解；难度即时保存；考点多选 +「保存考点」；题干只读 +「修改」；追问流式 |
| 侧栏 | `Sidebar.vue` 菜单项 |
| i18n | `wrongBook.*` / `preview.saveToWrongBook*` |

存入内容约定（实现默认）：

- `questionText`：存入时**服务端对原图 OCR**；客户端可不传。占位「见原题图片」/解题提示词会触发重抽。
- `answerText`：当前会话**最后一条 assistant** 的完整正文。
- `difficulty`：缺省 `medium`。
- `tags`：`string[]`，支持多选新建。
- 详情默认**刷题**；题解模式才渲染 `answerText`。
- 追问历史**默认不写入** `answerText`。
### 4.5 后端

按 `nest-backend-conventions`：

```text
src/wrong-questions/
  controllers/wrong-questions.controller.ts
  services/wrong-questions.service.ts
  services/wrong-questions-follow-up.service.ts  # 或同 service 内方法
  dto/
  wrong-questions.module.ts
```

- **复用：** `streamMathVisionChat`（`files/ai/vision/math-vision.provider.ts`）、StorageProvider、鉴权 Guard、F-27 的 MIME / `takeLastMessages`（可抽 shared util 或复制最小集，避免扯大重构）。
- **注册：** `app.module.ts` import `WrongQuestionsModule`。
- **不进** `document-index.processor`。

### 4.6 错误与边界

| 场景 | 行为 |
|------|------|
| `userFileId` 非本人 / 已删 | 创建 400/404 |
| 非图片文件 | 创建或 follow-up 400 |
| follow-up 时原图已删 | 400，文案「原图不可用」 |
| 流式中途客户端断开 | abort 上游 VL（对齐 F-27） |
| 详情追问气泡 | 仅本页内存；不自动 PATCH `answerText` |

## 10. F-29+ / 插件前置：`tempImageId`（本迭代不实现，接口先钉死）

> **目的：** 浏览器插件（网页截图 → 侧栏解题 → 存错题本）与「解题图默认不进网盘列表」依赖本节约定。  
> **排期：** F-29（Y1）完成之后单独立项；**禁止**用「浏览器只持 File、关页即丢」冒充正式方案。  
> **对齐：** [F-27 产品约定 · 临时上传中间态](./2026-07-18-F27-截图数学解题-产品约定.md)

### 10.1 产品行为

```text
插件/网页截图
  → POST 临时图 → 返回 tempImageId（TTL，默认 24h；可配置）
  → 解题 / 追问均用 tempImageId 取图（不写 UserFile，不出现在网盘列表）
  → 用户点「加入错题本」
       → 临时对象「转正」为网盘 UserFile（得到 userFileId）
       + 写 WrongQuestionBookItem（userFileId 非空）
  → 不点则 TTL 过期删除临时对象；之后解题/存入均 410/404
```

| 入口 | 解题身份 | 存入错题本 |
|------|----------|------------|
| 网盘预览（F-27 / F-29） | `fileId`（UserFile） | `POST /wrong-questions` + `userFileId`（已实现） |
| 插件 / 临时截图（F-29+） | `tempImageId` | `POST /wrong-questions/from-temp`（转正 + 写条目，一次完成） |

两条入口汇合后，列表/详情/二次追问（已落库的题）仍只认 `WrongQuestionBookItem.userFileId`。

### 10.2 API 形状（锁定，实现时勿随意改名）

| 方法 | 路径 | 请求 | 响应 | 说明 |
|------|------|------|------|------|
| POST | `/api/ai/math-temp-images` | `multipart` 图片（MIME 白名单同 F-27） | `{ tempImageId, expiresAt }` | 鉴权 JWT；对象写入**独立临时前缀**（非用户可见目录） |
| POST | `/api/ai/math-temp-images/:tempImageId/solve` | `{ question, messages? }` | **纯文本流** | 同 `streamMathVisionChat`；N=6；每次带该临时图 |
| DELETE | `/api/ai/math-temp-images/:tempImageId` | — | 204 | 可选：用户主动丢弃 |
| POST | `/api/wrong-questions/from-temp` | `{ tempImageId, questionText, answerText, tags?, fileName? }` | 创建后的 wrong-question item | **事务：** 转正 → UserFile → 写条目 → 删临时（或标记已消费）；幂等：同一 `tempImageId` 二次提交返回已有 item 或 409 |

约束：

- `tempImageId` 格式建议：`opaque` 字符串（UUID），**不可猜测**；校验 `userId` 归属。
- 过期 / 已消费 / 非本人 → `404` 或 `410 Gone`（实现选一，文档与前端统一）。
- **不做**「仅本地 File、服务端无图」的追问。
- 插件鉴权：复用站点登录 Cookie 或扩展持有的 Bearer；CORS / `host_permissions` 在插件立项时另定，不阻塞本 API。

### 10.3 存储与表（F-29+ 增量）

**临时对象（二选一，实现时选定并写回本节）：**

1. **推荐：** 表 `MathTempImage`（`id`/`tempImageId`、`userId`、`storageKey`、`mime`、`expiresAt`、`consumedAt?`）+ Storage 临时前缀。  
2. 备选：无表，仅对象存储 key = `math-temp/{userId}/{uuid}`，元数据放对象 header / Redis TTL。

**错题本表（F-29 已按可空 `userFileId` 建好则本步只加业务）：**

- 入库完成的条目：**必须**有 `userFileId`（转正后）。
- 不要把未转正的 `tempImageId` 长期写进 `WrongQuestionBookItem`（避免 TTL 删图后错题本悬空）；临时阶段图只活在 temp 子系统。
