# TTS 文字转语音 实现方案

> 日期：2026-07-22  
> 状态：**已完成（MVP）**；2026-07-23 增补方言/风格（含英语）  
> 关联：F-25 ASR 之后、`ai/` 目录重构之前；[F-25 ASR 设计](./2026-07-22-F25-音频转文字-ASR-设计.md)；[AI 目录重构里程碑](./2026-07-22-AI目录结构重构-里程碑.md)

## 1. 已确认的决策

### 1.1 UX 方案（用户选定）
- **选定：方案 C** — 划词朗读 + 独立「文字转语音」页；两套共用同一套合成能力。
- **权限（已修订）：** **仅「划词朗读」**由管理员按用户配置开关。管理员固定可用划词朗读；非管理员默认关闭。**独立「文字转语音」页登录即可用，不看此开关。**
- **用户主路径：**
  1. （需划词权限）文档/图片预览 → 划词 → 选音色 / 方言 →「朗读」→ 播放 / 下载  
  2. （全体登录用户）侧栏「文字转语音」→ 粘贴文本 → 选音色 / 方言 → 生成 → 试听 / 下载  
- **未选方案（归档）：** 仅 A（只有划词）；仅 B（只有独立页）。

### 1.2 技术方案（用户选定）
- **选定：路线 Y1（范围已收窄）** — `User.ttsEnabled` 只控制划词朗读入口；`can_use_tts` 下发给前端；`/api/ai/tts/*` **仅需登录**，不再按 `ttsEnabled` 拒绝。
- **关键技术点：** Prisma 字段；Admin 用户列表「划词朗读」开关；登录/`me` 下发 `can_use_tts`；硅基 CosyVoice speech；字数上限；方言/风格 instruct；MVP 不落库、不走 BullMQ。
- **未选路线（归档）：** Y2 仅全局总开关；Y3 用户开关 + 全局紧急总闸。

### 1.3 方言 / 风格（实现后增补）
- CosyVoice 默认按普通话读；粤语、英语等需通过 instruct 控制：  
  `用粤语说这句话<|endofprompt|>正文`
- 后端 `TTS_STYLE_PRESETS` + `buildTtsInput` 自动拼接；正文已含 `<|endofprompt|>` 时原样透传（用户自定义指令）。
- 前端独立页与划词朗读均提供「方言/风格」下拉。

| style id | instruct（空则直读） |
|----------|----------------------|
| `default` | （无，普通话直读） |
| `english` | `用英语说这句话` |
| `cantonese` | `用粤语说这句话` |
| `sichuan` | `用四川话说这句话` |
| `shanghai` | `用上海话说这句话` |
| `tianjin` | `用天津话说这句话` |

非法 / 缺省 `style` → 按 `default`。

## 2. 范围

### 2.1 In Scope（已落地）
- 后端 TTS Provider + HTTP（音色列表、方言列表、合成 mp3）
- `User.ttsEnabled` + Admin 开关 API + 列表展示（文案：划词朗读）
- `can_use_tts` 透出到前端（登录 / 拉取当前用户）
- 前端：`DocumentAiPanel` 划词朗读（仅划词模式、有选中、且 `can_use_tts`）
- 前端：独立页 `/tts`（侧栏入口，**全体登录用户**可见可进）
- `/api/ai/tts/*`：JWT 登录即可；**不**按 `ttsEnabled` 403
- Provider 单测（含粤语 / 英语 instruct 拼装）

### 2.2 Out of Scope
- 存进网盘、长文异步分段合并、自定义克隆音色、双人播客
- VIP 默认开通划词朗读（本期不绑 VIP）
- 对独立页再加配额/日限（可后续）
- 全局紧急总闸（Y3）
- `ai/` 目录大搬家（**ASR+TTS 已完成，见** [重构里程碑](./2026-07-22-AI目录结构重构-里程碑.md)）
- 文生视频

## 3. 用户操作对照

| 步骤 | 用户动作 | 界面反馈 | 失败/边界 |
|------|----------|----------|-----------|
| 1 | 管理员打开「用户管理」 | 「划词朗读」开关；管理员行固定开且不可关 | — |
| 2 | 管理员打开某普通用户开关 | 该用户刷新后预览可见划词朗读 | 不能关管理员 |
| 3 | 无划词权限用户打开 PDF 划词 | **不显示**朗读控件 | 仍可进独立 `/tts` 页 |
| 4 | 有划词权限用户划词 → 选方言 → 朗读 | 播放 / 下载 | 无选中、超字数提示 |
| 5 | 任意登录用户进侧栏「文字转语音」 | 选音色/方言后可合成试听/下载 | 超字数、空文本 |
| 6 | 未登录访问 `/tts` | 跳转登录 | — |
| 7 | 粤语 / 英语文本 + 对应方言 | 按该方言/语言朗读 | 选「普通话」则按普通话读 |

## 4. 技术设计

### 4.1 架构与数据流

```
[Admin「划词朗读」开关] → User.ttsEnabled
                         ↓
[登录 / me] → can_use_tts（仅控制预览划词朗读 UI）

[任意登录用户]
  → 侧栏 /tts 页 → GET/POST /api/ai/tts/*（只需 JWT）
  → DocumentAiPanel 仅当 can_use_tts 显示「朗读」

[合成]
  text + style → buildTtsInput → CosyVoice /audio/speech → audio/mpeg
```

`canUseTts` / `can_use_tts` 仍用于 Admin 列表与登录透出；**不再**用于拦截 `/ai/tts/*`。

### 4.2 API

| 方法 | 路径 | 请求 | 响应 | 鉴权 |
|------|------|------|------|------|
| GET | `/api/ai/tts/voices` | — | `{ success, data: { maxChars, voices, styles } }` | JWT（登录即可） |
| POST | `/api/ai/tts/speech` | `{ text, voiceId?, style?, speed? }` | `audio/mpeg`（`Content-Type: audio/mpeg`） | JWT（登录即可） |
| PATCH | `/api/admin/users/:id/tts` | `{ enabled: boolean }` | `{ success, data: { id, tts_enabled, can_use_tts } }` | JWT + `@Roles('admin')`；禁止关闭管理员 |

说明：
- `voices[]`：`{ id, label, gender }`（短 id，如 `alex`）
- `styles[]`：`{ id }`（与 `TTS_STYLE_PRESETS` 对齐）
- `GET /api/admin/users`（`listUsers`）含 `tts_enabled`
- 登录 `toUserDto` 与当前用户接口含 `can_use_tts`（派生字段；普通用户不能自改）

### 4.3 数据 / 存储
- **字段：** `User.ttsEnabled Boolean @default(false) @map("tts_enabled")`（migration `20260722123418_add_user_tts_enabled`）
- **复用：** 不写 Storage、不写 BullMQ
- **环境变量：**
  ```env
  AI_TTS_BASE_URL=https://api.siliconflow.cn/v1
  # AI_TTS_API_KEY=  # 空则回退 VISION → EMBEDDING → AI_API_KEY
  AI_TTS_MODEL=FunAudioLLM/CosyVoice2-0.5B
  AI_TTS_MAX_CHARS=2000
  ```

### 4.4 前端（已落地）
- `DocumentAiPanel/index.vue`：划词模式 + `canUseTts` → 音色 + 方言 +「朗读」+ `<audio>` + 下载
- `views/tts/index.vue`：独立工具页（音色 + 方言）
- `Sidebar.vue`：全体登录用户显示「文字转语音」（不看 `can_use_tts`）
- `admin/index.vue`：用户表「划词朗读」开关列
- `api/ai.ts`：`getTtsVoices` / `synthesizeSpeech({ text, voiceId?, style? })` → Blob
- `api/admin.ts`：`setUserTtsEnabled`
- `types/user.ts` + auth 映射：`canUseTts`
- 路由：`/tts`（需登录）
- i18n：`preview.aiTts*`、`ttsPage.*`（含 `style*`；zh-CN / zh-TW / en-US）

### 4.5 后端（已落地）
- `src/files/ai/tts/tts.provider.ts` — 音色 / 风格预设、`buildTtsInput`、`synthesizeSpeech`
- `src/files/ai/tts/tts.provider.spec.ts`
- `src/files/ai/tts/files-ai-tts.service.ts` / `files-ai-tts.controller.ts`（`@Controller('ai/tts')`）
- `src/files/ai/tts/tts-access.util.ts` — `canUseTts`（划词用）
- 已注册于 `files.module.ts`
- 相关修改：Prisma；`admin.service` / `admin.controller`；`auth.service`（`can_use_tts`）

### 4.6 错误与边界
- ~~无权限访问 `/ai/tts/*` 403~~（已取消；登录即可）
- 空文本 / 超字数：400
- 非法 `voiceId`：回退默认音色（`alex`）
- 非法 `style`：按 `default`（普通话直读）
- 上游失败：简短 message，风格对齐 ASR
- 对管理员 PATCH `enabled: false` → 400「管理员固定开放」

## 8. 风险与后续

- **风险：** 费用取决于调用量；字数上限默认 2000，日配额可后续加
- **风险：** `localStorage` 缓存用户信息，改划词开关后需刷新才更新 UI（可接受）
- **风险：** 方言效果依赖 CosyVoice instruct；个别音色/文本组合可能不稳定
- **明确延后：** 存网盘、长文队列、全局总闸、VIP 捆绑、`ai/` 重构、视频抽音轨
