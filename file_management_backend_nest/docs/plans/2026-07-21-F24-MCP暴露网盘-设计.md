## 1. 目标与非目标

**目标**

- 在 Cursor 里通过 MCP 工具浏览网盘目录、读取适合交给 LLM 的文本内容。
- 复用现有 Nest REST + JWT；MVP **不改**后端（类型策略在 MCP 侧拦截）。

**非目标（本期不做）**

- 听歌识曲、视频/音频二进制塞进模型
- 上传 / 删除 / 改权限等写操作
- 多用户配额、独立 MCP API Key 表
- `search_files` / `ask_document`（可二期叠加）
- 外挂 YouTube/B 站等外链

## 2. 已确认决策

| 项 | 选择 |
|----|------|
| 运行形态 | **A**：本机 MCP 进程（stdio）→ HTTP 调 Nest |
| 读内容策略 | **B**：txt/md 等可直读；pdf/docx/OCR 文稿须已抽取 |
| 工具范围 | **A**：仅 `list_files` + `get_file_text` |
| 用户模型 | 仅自己用；Token = 登录 JWT，配置在本机 `mcp.json` env |

## 3. 架构

```
Cursor (mcp.json)
  → node mcp-netdisk/dist/index.js   # MCP TypeScript SDK, stdio
    → Authorization: Bearer <JWT>
    → Nest API (localhost)
         GET /api/files
         GET /api/files/:id
         GET /api/files/:id/download
         GET /api/files/:id/ai/extracted-text
```

**仓库位置**

```
FileManagement_proj/
  mcp-netdisk/
    package.json
    src/index.ts         # MCP 入口、注册 tools
    src/api-client.ts    # Nest HTTP 客户端
    src/types-policy.ts  # 白/黑名单与可读性判断
    README.md
  file_management_backend_nest/   # MVP 零改动
```

**环境变量**

| 变量 | 含义 |
|------|------|
| `NETDISK_API_BASE` | 如 `http://127.0.0.1:3000` |
| `NETDISK_API_TOKEN` | 登录后的 access JWT（勿提交 git） |
| `NETDISK_MAX_CHARS` | 单次正文上限，默认 `50000` |

## 4. 工具契约

### 4.1 `list_files`

- **入参**：`parentId?`（省略=根）、`keyword?`（若列表接口支持则透传）
- **出参**（数组项）：`id`、`name`、`mimeType`、`isFolder`、`size`、`updatedAt`、`readable`
- 多媒体等不可读类型**仍列出**，但 `readable: false`，避免模型误读

### 4.2 `get_file_text`

- **入参**：`fileId`（必填）
- **出参**：`{ fileId, name, mimeType, truncated, text }`
- 超过 `NETDISK_MAX_CHARS` 截断，`truncated: true`，文末注明已截断

### 4.3 类型策略（`types-policy`）

| 类别 | 规则 |
|------|------|
| 直读白名单 | `text/*`、`application/json`、扩展名 `.md/.txt/.csv/.json/.log` 等 |
| 须索引 | `application/pdf`、docx、图片 → `GET .../ai/extracted-text` |
| 硬拒绝 | `video/*`、`audio/*`（可再加 zip/exe） |

### 4.4 错误文案（给模型看的短句）

| 情况 | 文案方向 |
|------|----------|
| 黑名单类型 | `不支持将该类型（…）交给 AI，请换文本或已索引文档` |
| 须索引但无正文 | `该文件尚未建立索引，请先在网盘点击「建立索引」后再试` |
| 文件夹 | `这是文件夹，请用 list_files 查看子项，或指定具体文件 id` |
| 401/403 | `网盘鉴权失败，请检查 NETDISK_API_TOKEN 是否过期` |
| 404 | `文件不存在或无权访问` |

## 5. Cursor 配置示例

```json
{
  "mcpServers": {
    "netdisk": {
      "command": "node",
      "args": ["F:/code/FileManagement_proj/mcp-netdisk/dist/index.js"],
      "env": {
        "NETDISK_API_BASE": "http://127.0.0.1:3000",
        "NETDISK_API_TOKEN": "<access JWT>",
        "NETDISK_MAX_CHARS": "50000"
      }
    }
  }
}
```

Token：浏览器登录或 `POST /api/auth/login` 取 `accessToken`；过期后手动更换。

## 6. 安全（单人最低线）

- 含 Token 的本地配置不进 git（`.gitignore` + README 警示）
- MVP 工具全部只读
- 不做「把任意二进制 base64 塞进上下文」
