# F-25 音频转文字（ASR）— 实现方案

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.  
> 日期：2026-07-21  
> 状态：📝 待实现  
> 关联 PRD：[2026-07-08-ai-features-prd.md](./2026-07-08-ai-features-prd.md) § F-25 / §1.5 **段 4**  
> 定位：里程碑第 4 段核心能力；与 **段 2 视频总结** 共用 ASR 基建（视频多一步「抽音轨」）

**Goal：** 用户对网盘内 `audio/*` 文件点「建立索引 / 转写」，异步得到带时间戳的转写文稿，可在预览中查看；可选再生成结构化摘要（段 4 完整形态）。

**Architecture：** 复用现有 `document-index` BullMQ 流水线。音频路径：`Storage 读文件 →（可选）ffmpeg 转成 ASR 友好格式 → 云 Whisper 兼容 ASR（长音频分段）→ 按时间窗写入 DocumentChunk（含 startMs/endMs）→ embedding → ready`。摘要走既有 Map-Reduce（可选 Phase B）。**不做**识曲、分轨、外链下载。

**Tech Stack：** NestJS、BullMQ Worker、Prisma/MySQL、`fluent-ffmpeg` + `@ffmpeg-installer/ffmpeg`（项目已有）、云 ASR（OpenAI Whisper 兼容，推荐硅基）、Vue3 预览面板。

**工期粗估：** 仅转写 MVP **约 3～5 天**；含摘要/RAG 对齐 F-25 段 4 则 **约 2～4 天**（若 ASR 已在视频段落地，音频扩展往往只要 2～3 天）。

## 0. 范围与分期

### 0.1 本期做

| 项 | 说明 |
|----|------|
| 输入 | 网盘已上传 `audio/*`（mp3 / wav / m4a / ogg 等；上传类型不人为再砍） |
| 触发 | 复用 `POST /api/files/:id/ai/index`（扩展可索引 MIME） |
| 输出 | 带 `startMs`/`endMs` 的 `DocumentChunk`；`GET .../ai/extracted-text` 或专用 transcript API 可读全文 |
| 失败 | 超时长、无有效语音、ASR 失败 → `failed` + 明确中文错误 |
| 边界 | 纯音乐可「未识别到可用语音」失败，不硬撑歌词质量 |

### 0.2 本期不做

- 听歌识曲 / 伴奏分离 / 音乐生成  
- YouTube/B 站外链  
- 本地 Whisper GPU 部署（二期）  
- 文字转音频（TTS）、文生视频（已否决）

### 0.3 与视频段的关系（重要）

```
共享层：asr.provider + media 预处理 + chunk 时间戳 + Worker 分支
    ↑
段 2 视频：ffmpeg 抽音轨 → 共享层 →（摘要 + 播放器跳转）
段 4 音频：音频文件直接进共享层 →（摘要 + 播放器/文稿面板）
```

**推荐实施顺序（与 PRD §1.5 一致）：** 先落地共享 ASR + 视频抽轨（段 2），再给 `audio/*` 开索引白名单（段 4，增量小）。

**若只想先做「音频转文字」：** 仍按本文 Task 建共享 ASR；视频抽轨可标为后续 Task，不阻塞音频路径。

### 0.4 Phase 划分

| Phase | 交付 | 建议 |
|-------|------|------|
| **A — 转写 MVP** | 音频 → ASR → chunks + 文稿展示 + 索引进度 | **先做** |
| **B — 摘要** | `audio_overview` / chapters；复用 F-03 Map-Reduce | 段 4 完整验收 |
| **C — RAG（可选）** | 对文稿划词/全文问答（复用 F-02） | 可与 B 同期或略后 |

## 1. 数据流

```
用户点击「转写/建立索引」
  → FilesAiIndexService 校验 audio MIME，入队 document-index
  → DocumentIndexProcessor
       progressMsg: 正在准备音频…
       → 读 Storage → 本地临时文件
       →（若非 16k mono wav）ffmpeg 转码
       → 按时长切段（建议 ≤10～15 分钟/段，受 ASR API 限制）
       → 逐段 asr.transcribe(file) → segments[{ text, startMs, endMs }]
       → merge → chunkText 或按时间窗切 DocumentChunk
       → embedMany → ready
  → 前端轮询 index/status；预览展示文稿时间轴
```

### Task 2: 媒体预处理（ffmpeg）

**Files:**
- Create: `src/files/ai/asr/media-audio.util.ts`
- Create: `src/files/ai/asr/media-audio.util.spec.ts`（可 mock fluent-ffmpeg）

## 5. 验收标准

### Phase A（转写）

- [ ] `audio/mpeg`（≤30 分钟样例）索引 → `ready`
- [ ] chunks 含非空 `content` 与合理 `startMs`/`endMs`
- [ ] 前端可见文稿；点击可跳转播放
- [ ] 同文件二次索引（hash 未变）被拒绝或短路
- [ ] 超时长 / ASR 失败 → `failed` + 可读错误
- [ ] 纯音乐或无语音：明确失败或「未识别到可用语音」（产品可选其一，需统一）

### Phase B（段 4 完整）

- [ ] 结构化摘要入库；章节可跳转
- [ ] （可选）RAG 能基于文稿回答

## 6. 风险与缓解

| 风险 | 缓解 |
|------|------|
| ASR 计费/限流 | 时长上限；分段；VIP 配额（F-18 未做前先硬上限） |
| 长音频临时磁盘 | tmp 分段处理 + finally 删除 |
| 供应商 segments 格式不一致 | provider 内归一化；单测多 fixture |
| 视频未做时重复造轮子 | 本方案强制「共享 asr/」目录，视频只加抽轨 |
