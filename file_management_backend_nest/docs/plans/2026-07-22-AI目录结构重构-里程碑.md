# AI 目录结构重构（里程碑）

> 日期：2026-07-22  
> 状态：⏳ 待办（**ASR + TTS MVP 已完成**，可启动本里程碑；完成后再开视频抽音轨）  
> 范围：`src/files/ai/` 目录整理，**以搬家为主、不改业务语义**  
> 关联：F-25 ASR；[TTS 实现方案](./2026-07-22-TTS-文字转语音-实现方案.md)；视频抽音轨延后到本里程碑之后

---

## 1. 为什么做

当前 `ai/` 根目录混放了：

- HTTP：`files-ai.controller.ts`、`image-search.controller.ts`
- 应用服务：大量 `files-ai-*.service.ts`
- 异步 Worker：`document-index.processor.ts` + queue 相关
- 与按能力拆的子目录（`asr/`、`embedding/`、`vision/`、`chunk/`…）平铺在一起

阅读成本高，新人难以判断「该改哪一层」。重构目标是对齐 `nest-backend-conventions`：**按功能域分子目录**，根上只留模块装配与必要入口。

### 分层约定（重构后必须遵守）

| 层 | 放什么 | 不放什么 |
|----|--------|----------|
| `http/` 或各域 `*.controller.ts` | 路由、DTO 校验、调 service | 重业务、直接 ffmpeg/ASR |
| 各域 `*.service.ts` | 鉴权边界、Prisma、入队、编排 | 供应商 HTTP 细节可下沉 Provider |
| `*.processor.ts` | BullMQ 长任务流水线 | 对外 REST |
| `asr/` / `embedding/` / `vision/` / `chunk/` / `tts/` | Provider、纯函数、单测 | 尽量不注入 Prisma（除非明确例外） |

`*.spec.ts`：**继续与实现文件同目录**（本仓库惯例）。

---

## 4. 执行原则

1. **只搬家、改 import、改 module providers 注册路径**；不借机改业务逻辑。  
2. 一次 PR 可按域分批（例如先 `index/`，再 `chat/`），每批跑相关 jest + 手测一条索引/问答。  
3. 禁止两层及以上相对路径 `../../`，跨文件用 `@/`（见 nest 规范）。  
4. 重构期间不新增视频抽音轨代码进乱目录；视频需求在里程碑③单独开设计。
