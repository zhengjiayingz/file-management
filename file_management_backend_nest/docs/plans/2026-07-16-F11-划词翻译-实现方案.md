## 1. 已确认的决策

### 1.1 UX 方案（用户选定）

- **选定：方案 A（增强）** — 复用右侧 AI 问答面板；在面板头部（标题旁）放置：
  - **目标语言下拉**（含「默认」）
  - **翻译按钮**
- **可见性（已确认）：** 翻译按钮与目标语言下拉 **仅在「划词问答」模式显示**；切换到「全文问答」时 **隐藏**（不做整篇翻译）。
- **用户主路径：** 左侧划选文字 → 确保处于「划词问答」→（可选）选目标语言 → 点「翻译」→ 译文以**流式气泡**进入问答历史（与划词问答同一套对话区，可停止、可继续追问）。
- **默认语言规则：**
  - 下拉为「默认」时：选中文本**主要是中文** → 译成英文；否则 → 译成中文。
  - 用户选择了具体目标语言（中文 / 英文 / 日文）→ **以下拉为准**，不再走自动判定。
- **未选方案（归档）：**
  - 方案 B 选区旁浮动条 — 工期长、与 PDF/虚拟列表划词冲突风险高。
  - 方案 C 仅预览顶栏按钮 — 离选中上下文远，体验弱于面板内操作。
  - 全文问答模式下仍显示翻译 / 翻译整篇 — 否决；改为隐藏入口。

### 1.2 技术方案（用户选定）

- **选定：路线 Y1 + 流式对话体验**
  - 独立接口 `POST /api/files/:id/ai/translate`。
  - **后端**解析 `targetLang: 'default' | 'zh' | 'en' | 'ja'`（默认时做中文占比判定 → zh/en，**默认逻辑不含日文**）。
  - 响应形态对齐现有 `ai/ask`：**`streamText` 流式 `text/plain`**（或与 ask 相同的 pipeTextStream）。
  - 前端复用 `DocumentAiPanel` 聊天列表：写入一条用户消息 + 一条流式助手消息，进入 `chatMessages` 历史。
- **关键技术点：** Nest AI Service、`getChatModel` + `streamText`、汉字占比检测、前端流式 fetch（复用 `api/ai.ts` 模式）、i18n、`v-if="chatMode === 'selection'"` 控制入口显隐。
- **未选路线（归档）：**
  - Y2 仅前端判定语言 — 多端规则易不一致。
  - Y3 纯复用 ask、无「翻译」当 question — 弃用；改为专用 translate API，但 UI/流式体验与 ask 一致。
  - 初版 Y1「非流式 JSON」— 按用户反馈改为流式进历史。

## 2. 范围

### 2.1 In Scope

- **仅划词问答模式**展示翻译下拉 + 按钮；全文问答模式隐藏。
- 有选中文本时可点「翻译」。
- 目标语言：`默认` / `中文` / `英文` / `日文`。
- 后端默认译向判定（中↔英）+ 指定语言（含日文）。
- 流式输出进问答历史；支持停止（复用现有 Abort）。
- 超长选中文本截断提示（对齐 ask 的 8000 上限）。
- 文案三语：zh-CN / zh-TW / en-US。
- PDF / 文本 / Word（已挂 DocumentAiPanel）一并可用，无需改各预览弹窗入口（只改面板）。

### 2.2 Out of Scope

- 选区旁浮动工具条。
- 法/德等更多目标语言（日文已纳入；其余可后续加枚举）。
- F-18 限流 / VIP 配额（翻译接口预留同鉴权即可）。
- 翻译结果单独持久化表（仅前端会话内历史；关弹窗即清，与现 ask 一致）。
- **全文问答模式下的翻译入口或整篇翻译**（隐藏按钮，不做）。

## 4. 技术设计

### 4.1 架构与数据流

```
（仅 chatMode === 'selection' 时 UI 可见）
用户点「翻译」
  → 前端解析下拉 targetLang（default|zh|en|ja）
  → POST /api/files/:id/ai/translate { text, targetLang, fileName? }
  → 后端：鉴权 + 文件归属
  → 若 targetLang=default：isMostlyChinese(text) ? en : zh
  → 若 targetLang 为 zh|en|ja：用指定语言
  → streamText(翻译 system prompt + 原文)
  → pipeTextStreamToResponse
  → 前端 onChunk 写入 chatMessages 助手气泡
```

### 4.2 API

| 方法 | 路径 | 请求 | 响应 | 鉴权 |
|------|------|------|------|------|
| POST | `/api/files/:id/ai/translate` | `{ text: string, targetLang: 'default' \| 'zh' \| 'en' \| 'ja', fileName?: string }` | **流式文本**（与 `ai/ask` 相同：`pipeTextStreamToResponse`） | 登录 JWT（现有 Files 鉴权） |

说明：

- **不传**多轮 `messages`（翻译是单次任务）；译完后的追问仍走 `ai/ask`，前端把翻译气泡留在 `chatMessages` 里即可。
- 错误未开流前：`400/404/500` + `{ success:false, message }`，与 ask 一致。

### 4.3 数据 / 存储

- 新表或字段：**无**
- 复用：`userFile` 归属校验；无会话表

### 4.4 前端

**入口组件：** `file_management_frontend/src/components/DocumentAiPanel/index.vue`

- Header 右侧：目标语言 `el-select` + 「翻译」`el-button`，外包一层 **`v-if="chatMode === 'selection'"`**（全文问答时整组隐藏）。
- 点击翻译（仅划词模式可达）：
  1. 校验 `selectedText`；
  2. 保持 `rightPanelTab = 'chat'`、`chatMode = 'selection'`（无需从全文模式切回）；
  3. push 用户消息（文案示例：`翻译 → 中文` / `翻译 → 英文` / `翻译 → 日文` / `翻译 → 默认（自动）`）；
  4. push 助手流式消息；
  5. 调用 `streamTranslate(...)`。
- 进行中复用 `asking` / `askAbort` / 「停止」按钮（翻译中不可同时发送普通提问）。

**新增/修改文件：**

| 文件 | 变更 |
|------|------|
| `src/components/DocumentAiPanel/index.vue` | 下拉、按钮（仅划词可见）、`submitTranslate` |
| `src/api/ai.ts` | `streamTranslate`（照 `streamAskAboutSelection` 抄流式 fetch） |
| `src/locales/zh-CN.ts` / `zh-TW.ts` / `en-US.ts` | 按钮、下拉（含日文）、错误、用户气泡文案 |

**建议 i18n key（实现时可微调）：**

- `preview.aiTranslate` / `preview.aiTranslateTarget` / `preview.aiTranslateDefault`
- `preview.aiTranslateZh` / `preview.aiTranslateEn` / `preview.aiTranslateJa`
- `preview.aiTranslateNoSelection` / `preview.aiTranslateTruncated`
- `preview.aiTranslateUserMsg`（带目标语言插值）

### 4.5 后端

| 项 | 路径 |
|----|------|
| Controller | `src/files/ai/files-ai.controller.ts` — 新增 `@Post(':id/ai/translate')` |
| Service | **推荐新建** `src/files/ai/files-ai-translate.service.ts`（避免 `files-ai.service.ts` 继续膨胀）；或在同文件加方法，二选一实现时以新建 Service 为准 |
| 语言工具 | `src/files/ai/utils/detect-chinese.util.ts` — `isMostlyChinese(text): boolean`（汉字占比阈值，建议 ≥ 0.3 或按字母/汉字计数比） |
| 复用 | `getChatModel`、`clampText`、文件归属查询、`AbortController` + `req.on('close')` |

**Prompt 要点（system）：**

- 只输出译文，不要解释、不要加「译文：」前缀；
- 目标语言写死为 resolved 后的语言名（中文 / English / 日本語）；
- 保留专有名词合理处理；
- 原文不足或为空则短提示。

**Module 注册：** `src/files/files.module.ts` 若新建 Service 则 providers/exports 补上。

### 4.6 错误与边界

| 情况 | 处理 |
|------|------|
| 未选中文本 | 前端拦截；后端 400「请先选中要翻译的内容」 |
| `targetLang` 非法 | 400（合法：`default` \| `zh` \| `en` \| `ja`） |
| 文本过长 | `clampText` 到 8000；前端可先提示已截断 |
| 中英混排 + 默认 | 按汉字占比阈值；不追求完美语种识别 |
| 用户中断 | Abort → 前端保留已流出的部分译文 |
| 全文问答模式 | **不展示**翻译控件；无「切回划词再译」的强制逻辑 |
