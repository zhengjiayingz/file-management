# F-25 音频转文字 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 音频文件走现有索引流水线，ASR 转写为带时间戳的 chunk，前端可悬停高亮、点击跳播。

**Architecture:** 复用 document-index BullMQ；音频分支 = ffmpeg 预处理 → 云 ASR → DocumentChunk(startMs/endMs) → embedding；前端读 transcript API。

**Tech Stack:** NestJS、Prisma/MySQL、fluent-ffmpeg、硅基 SenseVoice（Whisper 兼容）、Vue3 音频预览。

**设计文档:** `docs/plans/2026-07-22-F25-音频转文字-ASR-设计.md`

---

### Task 1: Prisma 时间戳字段

**Files:**
- Modify: `prisma/schema.prisma`（`DocumentChunk`）
- Create: `prisma/migrations/20260722120000_add_document_chunk_timestamps/migration.sql`

**Step 1: 在 DocumentChunk 增加字段**

在 `content` 后增加：

```prisma
  /// 音频转写句起始/结束（毫秒）；普通文档为 null
  startMs     Int?                  @map("start_ms")
  endMs       Int?                  @map("end_ms")
```

**Step 2: 手写 migration.sql**

```sql
ALTER TABLE `document_chunks` ADD COLUMN `start_ms` INTEGER NULL;
ALTER TABLE `document_chunks` ADD COLUMN `end_ms` INTEGER NULL;
```

**Step 3: 应用并生成 Client**

```powershell
cd F:\code\FileManagement_proj\file_management_backend_nest
pnpm exec prisma migrate deploy
pnpm exec prisma generate
```

若 `generate` EPERM：先停 API/Worker 再执行。

**Step 4: 验证**

- DB 表 `document_chunks` 有 `start_ms` / `end_ms`
- 现有文档索引仍可用
