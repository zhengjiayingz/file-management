**Goal：** 网盘图片预览一键 VL 流式解题，右侧独立会话可追问，助手气泡支持 KaTeX。

**Architecture：** 服务端读网盘图 → data URL → 独立 `AI_MATH_*` 视觉模型 `streamText`（多模态 messages）→ 与 ask 相同的纯文本流；**不进** index Worker。前端 `DocumentAiPanel` 增加 `solve` 模式与独立消息列表；`renderMarkdown` 接入 KaTeX（流式容错、结束全量重渲）。

**Tech Stack：** NestJS、Vercel AI SDK `streamText`、`@ai-sdk/openai`、StorageProvider、Vue3、`marked` + KaTeX、现有 `api/ai.ts` 流式 fetch。

| 步骤 | 用户动作 | 界面反馈 | 失败/边界 |
|------|----------|----------|-----------|
| 1 | 打开网盘图片预览 | 顶栏可见「解题」 | — |

| 7 | 关预览 Dialog | 全部会话销毁 | — |
| — | 解题中再点「解题」 | **重开**：清空 `solveMessages` 并再发首轮（实现约定） | 若正在 streaming 先 abort |

## 4. 技术设计

### 4.1 架构与数据流

```
顶栏「解题」/ 追问提交
  → POST /api/files/:id/ai/solve-math
       { question, messages?: [{role,content}], fileName? }
  → 鉴权 + UserFile 归属 + MIME ∈ 图片白名单
  → StorageProvider 读 buffer → data:${mime};base64,...
  → messages' = takeLast(messages, 6)
  → streamText({
       model: getMathVisionModel(),
       system: 解题 system prompt,
       messages: [
         ...historyAsText,
         { role:'user', content: [
             { type:'image', image: dataUrl },  // 或 image_url 按 SDK 约定
             { type:'text', text: question }
         ]}
       ]
     })
  → pipeTextStreamToResponse
  → 前端 onChunk 写入 solveMessages；streaming=false 时全量 renderMarkdown
```

**说明：** 历史轮次只发 **文字**；**每一轮当前 user 消息都附带原图**（产品约定方案 3）。服务端每次请求重新读盘（MVP 简单；追问频繁时可后续加短缓存，非本期）。

### 4.2 API

| 方法 | 路径 | 请求体 | 响应 | 鉴权 |
|------|------|--------|------|------|
| POST | `/api/files/:id/ai/solve-math` | 见下 | 流式 `text/plain`（同 ask） | JWT |

```ts
// 请求体
{
  question: string           // 必填；首轮前端可固定文案
  messages?: AiChatMessage[] // 可选；服务端再截断为最近 6 条
  fileName?: string
}
```

**错误：**

| 情况 | HTTP |
|------|------|
| body / question 无效 | 400 |
| 文件不存在或不属于用户 | 404 |
| 非图片 MIME / 不支持扩展名 | 400（文案说明仅支持 png/jpg/jpeg/webp/gif） |
| 模型/读盘失败 | 500（headers 未发送时） |

**不提供** 本期 `POST /api/ai/solve-math-image`（临时上传），避免与后续 `tempImageId` 设计冲突；预留注释即可。

### 4.3 后端模块

| 文件 | 职责 |
|------|------|
| `src/files/ai/vision/math-vision.provider.ts` | `getMathVisionModel()`：读 `AI_MATH_VISION_BASE_URL` / `API_KEY` / `MODEL`；缺省可回退硅基 URL，**Key 不与 OCR 强绑定**（可用独立 Key） |
| `src/files/ai/files-ai-math.service.ts` | 校验、读图、截断历史、system prompt、`streamText`、pipe |
| `src/files/ai/files-ai-math.service.spec.ts` | 校验与截断单测（mock model / prisma / storage） |
| `src/files/ai/files-ai.controller.ts` | 注册 `POST :id/ai/solve-math` |
| `src/files/files.module.ts` | 注册 `FilesAiMathService` |
| `.env.example` | 取消注释并写清 `AI_MATH_VISION_*`（已有草稿则对齐） |

**图片白名单：** 与 F-26 对齐 — `png/jpg/jpeg/webp/gif`（扩展名 + `mimeType.startsWith('image/')` 双检）。

**读图：** 注入现有 `StorageService`（或与 index 相同的 getStorageProvider），`exists` + `getReadStream` → Buffer（可抽小函数 `readStorageToBuffer`，避免复制大段逻辑）。

**System Prompt 要点（实现时写入常量）：**

- 学习辅助，分步推理；公式用 `$` / `$$` LaTeX  
- 看不清的条件标注不确定，禁止伪造  
- 结尾可提示用户自行验算  
- 优先中文作答  

**Abort：** 与 translate 相同 — `req.on('close')` → `abortController.abort()`。

**历史截断：**

```ts
export const SOLVE_MATH_HISTORY_LIMIT = 6

export function takeLastMessages(
  messages: ChatMessage[],
  limit = SOLVE_MATH_HISTORY_LIMIT,
): ChatMessage[] {
  if (messages.length <= limit) return messages
  return messages.slice(-limit)
}
```

### 4.4 前端

| 文件 | 职责 |
|------|------|
| `src/api/ai.ts` | `streamSolveMath({ fileId, question, messages, signal, onChunk })` |
| `src/utils/renderMarkdown.ts` | 接入 KaTeX（如 `marked-katex-extension` 或自定义 renderer）；`throwOnError: false` |
| 全局样式 | 引入 `katex/dist/katex.min.css`（main.ts 或组件侧） |
| `DocumentAiPanel` | `chatMode` 增加 `'solve'`；`solveMessages` 独立；`enableSolveMode` prop（仅图片预览传 true）；解题 UI：提示条 + 输入框；**隐藏**划词翻译入口 |
| `ImageDocumentPreviewDialog` | 顶栏「解题」按钮 → `aiPanelRef.startSolve()` 或 emit/provide |
| i18n | `preview.solveMath*`、`preview.aiChatModeSolve`、验算提示 |

**模式切换逻辑（要点）：**

```text
startSolve():
  abort 进行中的 ask/rag/translate/solve
  chatMessages = []          // 清空划词/RAG
  chatMode = 'solve'
  若已有 streaming solve → abort
  solveMessages = []         // 再点「解题」= 重开
  追加 user「请分步解答本题」+ assistant streaming
  streamSolveMath(...)

切到 selection/rag：
  chatMode 切换；不清 solveMessages

关 Dialog：
  组件销毁 → 全部清空
```

**KaTeX 流式策略（方案 B）：**

- 助手气泡始终绑定 **原始 Markdown 字符串** `msg.content`  
- `v-html="renderMarkdown(msg.content)"`（内部 try/catch 或 `throwOnError: false`）  
- `streaming === true` 时半截公式可失败显示原文  
- `streaming` 置 false 后同一 `content` 再渲 → 最终正确  

### 4.5 多模态 messages 注意

Vercel AI SDK + OpenAI 兼容提供商对 image part 字段名可能为 `image` / `image_url`。实现时以 `@ai-sdk/openai` 当前文档为准，在 `math-vision.provider` 或 service 内集中构造，单测 mock `streamText` 断言 messages 含图片 part。

若所选硅基 VL 模型不支持 SDK 封装，可退化为与 `vision.provider` 类似的 **SSE fetch 自管流**，但仍保持响应形态为纯文本流，前端无需两套解析。（优先尝试 SDK。）
