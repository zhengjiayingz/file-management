## MVP 范围（本期）

**只做网盘图片解题**：用户已将截图存入网盘 → 打开 `ImageDocumentPreviewDialog` → 顶栏「解题」。

| 做 | 不做（本期） |
|----|----------------|
| `POST /api/files/:id/ai/solve-math`（`fileId`） | 临时上传 `POST /api/ai/solve-math-image` |
| 右侧独立「解题」会话；可追问 | 「加入错题本」落库（→ F-29） |
| KaTeX 公式渲染（流式容错，结束全量重渲） | 像素框选局部解题 |
| 文案：「AI 辅助，请验算」 | F-18 硬限流（开放/上线前再补） |

## 已对齐的交互约定

1. **入口**：图片预览顶栏「解题」；非图片预览不出现。  
2. **会话**：右侧独立解题线程；切入时 **清空** 划词 / RAG 历史。  
3. **追问**：支持；每轮 **带原图** + 最近 **N=6** 轮文字消息。  
4. **切回划词/RAG**：不清解题历史；**关掉预览 Dialog 才清**（`destroy-on-close` 自然丢）。  
5. **渲染**：升级 `renderMarkdown`（`marked` + KaTeX）；流式中允许半截公式乱/失败，`streaming=false` 时用完整 Markdown **强制再渲一次**。  
6. **后端**：独立 VL 服务 / env（`AI_MATH_*`），**不进** `document-index.processor`。

## 后续必做：临时上传「中间态」（记一笔）

最终产品要支持：**解题默认不进网盘列表**；用户点「加入错题本」才把原题图落库。

推荐形态（F-29 或 F-27 增强期落地）：

```text
临时上传 → 服务端签发 tempImageId（TTL，如 24h）
         → 解题 / 追问均用 tempImageId 取图（不写 UserFile）
         → 用户点「加入错题本」
              → 临时对象「转正」为网盘文件（userFileId）
              + 写 WrongQuestionBookItem
         → 不点则 TTL 过期删除
```

| 要点 | 说明 |
|------|------|
| 为何需要中间态 | 发给模型后若立刻丢图，「加入错题本」时无法再落原题图 |
| 网盘图路径 | 已有 `fileId`，加入错题本只关联即可，无需转正 |
| 与 F-27 MVP | **本期不实现**临时上传与 tempId；API / 表结构预留时勿堵死该路径 |
| 与 F-29 | 「加入错题本」按钮 + 临时转正 + 持久化题干/解答；网盘题与临时题两条入口汇合 |

**明确不做的简化**：浏览器仅持有 File、关页即丢且无法服务端追问取图——仅适合极简 Demo，正式中间态用 **服务端 tempImageId**。
