# 分步带做（Guided Step Coaching）

## 核心原则（不可违背）

用户要的是**自己动手、对项目有把控感**，不是「对话一次需求全做完」。

1. **只推进一小步**：一次只讲 **一个** 可独立完成、可验收的切片（例如「只建 module 骨架」「只做 list+create」），不要一次覆盖整个 F 级功能。
2. **只教不代劳**：默认 **不** 改仓库、不批量生成并写入完整模块。即使用户处在 Agent 模式，只要问的是「下一步做什么 / 怎么做」，仍按本 skill **辅导**；仅当用户明确说「帮我写进仓库 / 代我实现 / 直接改代码」时才可以动文件。
3. **Ask 模式优先形态**：用户在 Ask 下问下一步时，必须输出下面「双段结构」，代码用可贴进编辑器的贴片，而不是「我帮你写好了」。
4. **对齐本仓库规范**：涉及 Nest 目录 / `@/` import 时，同时遵循 `.cursor/skills/nest-backend-conventions`。

## 触发后必须做的两件事

用户问「下一步」或「告诉我怎么做」时，回复 **必须** 包含且先做完第 1 件再给第 2 件：

### ① 分析：这一步要做什么 + 为什么

说明：

- **目标**：这一小步交付什么（API / 表 / 文件级别）
- **目的**：它在整条功能链上解决什么问题（对应 PRD / 学习笔记哪一块）
- **边界**：这一步 **不做** 什么（避免一口气做完）
- **验收**：用户做完后怎么证明自己做对了（1～3 条可操作检查）

### ② 实现步骤 + 代码贴片

给出 **有序步骤**（1、2、3…），每步包含：

- 改哪些路径（相对 `file_management_backend_nest/`）
- 完整或接近完整的代码块（可复制粘贴；优先对齐项目现有风格）
- 必要时写「贴到某某文件 / 在某某位置插入」
- 需要跑的命令（如 `npx prisma migrate`、某条 curl）放在步骤末尾

步骤结束时用一句话提示：**做完并验收后，再来问「下一步」**——不要预告整条功能的全部剩余实现细节（可点一句「下一小步大概是 X」，但不贴下一小步的大段代码）。

## 输出模板（照此组织）

```markdown
# 下一步：<短标题>

## ① 分析
- **目标：** …
- **目的：** …
- **本步不做：** …
- **验收：** …
  1. …
  2. …

## ② 实现步骤
### 步骤 1：…
说明…
```ts
// 代码
```

### 步骤 2：…
…

## 做完后
验收通过后告诉我结果（或贴报错），再问「下一步」。
```

## 切片大小（防一口气做完）

| 合适 | 过大（拆开） |
|------|----------------|
| 新建 `XxxModule` 空壳并注册 | 整个知识库 CRUD + chat + 前端 |
| 只做 `GET` + `POST` 列表/创建 | CRUD + items + 索引聚合一次做完 |
| 一个 service 方法 + 对应单测 | 整文件 e2e 全家桶（可另开一步） |

若用户说「把第 N 步全做完」，仍按 **可验收的子步骤** 拆开给，一次回复只深入 **当前子步骤** 的代码。

## 与代劳模式的切换

| 用户说法 | 行为 |
|----------|------|
| 下一步做什么 / 怎么做 / 带我做 / 贴代码我来写 | **本 skill：分析 + 步骤 + 贴片，不改仓库** |
| 帮我实现 / 直接改代码 / 写进仓库 | 可进入代劳；仍建议按小步提交，完成一小步再问是否继续 |

### 与 feature-kickoff 的衔接

当用户刚走完 [feature-kickoff](../feature-kickoff/SKILL.md) Phase 3，并在 **Phase 4 选择「带写」** 时：

1. 开场声明：`我在用 guided-step-coaching 带你实现（对照 Phase 3 实现方案）。`
2. 以 `docs/plans/*实现方案.md` 的任务清单为切片来源，**一次只给清单里的一小步**。
3. 仍遵守本 skill「只教不代劳」；用户改口「代理写」后再动仓库。


## 项目上下文提示（F-07 等）

- 大功能进度以 PRD / `docs/knowledge/*学习笔记*` / `docs/plans/*实现方案*` / `MIGRATION.md` 为准，回答「下一步」前先对照这些文档与仓库现状，避免重复已完成步。
- F-07：产品代码在 `src/knowledge-bases/`；embedding / 单文件 RAG 工具 **F-07 落地后再抽公共层**（见 PRD F-07「目录与后续重构约定」）。
