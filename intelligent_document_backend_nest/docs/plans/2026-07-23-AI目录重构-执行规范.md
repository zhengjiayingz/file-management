## 1. 目标与边界

### 1.1 做什么

- 按**应用域**整理 `ai/` 目录树。
- 域内按**文件类型**分子目录（controller / service / provider / utils / types / processor / test）。
- 原「能力层」目录（`asr`、`chunk`、`embedding`、`vision`）及顶层 `dto`、`utils` **并入宿主应用域**，不再与应用域平级。
- 修改 `@/` import 与 Nest `files.module` / `preview-worker.module` 等注册路径。

### 1.2 明确不做

- 不改业务逻辑、API 契约语义、队列名、class 名（除非为路径所必需且行为不变）。
- 不做视频抽音轨、TTS/ASR 新功能、依赖大升级。
- 不借机「优化算法」或大规模重写 Provider。
- 不引入 `ai/shared/` 或其它与应用域平级的杂项目录。

## 2. 目录硬约束

### 2.1 `ai/` 直接子级 = 应用域目录 + 根上 controller

**应用域目录**（名称固定）：

| 应用域 | 职责 |
|--------|------|
| `index/` | 建索引、Worker、以及索引管线能力（ASR / 切块 / OCR / 文本向量） |
| `chat/` | 划词问答、RAG、会话落盘、LLM 公共工具、聊天 DTO |
| `summary/` | 分层摘要 |
| `knowledge/` | 学术知识卡片 |
| `math/` | 截图解题（含 math-vision） |
| `translate/` | 划词翻译（含 detect-chinese） |
| `image-search/` | 以图搜图（含图像向量、fingerprint） |
| `assistant/` | F-42 网盘 AI 助手 |
| `tts/` | 文字转语音 |

**Controller** 一律放在 `ai/` **根上**（与应用域目录平级），列在一起，例如：

- `ai/files-ai.controller.ts`
- `ai/image-search.controller.ts`
- `ai/files-ai-tts.controller.ts`
- `ai/files-ai-assistant.controller.ts`

**禁止**再出现与应用域平级：

`http/` · `asr/` · `chunk/` · `embedding/` · `vision/` · `dto/` · 顶层 `utils/` · `fingerprint/` · `shared/` · 根上散落的 `files-ai-*.service.ts`

### 2.2 域内类型子目录（每个应用域统一）

| 位置 | 放什么 | 不放什么 |
|------|--------|----------|
| **`ai/` 根** | **全部** `*.controller.ts`（AI 路由入口，列在一起） | service / provider / util / spec |
| **各应用域顶层** | 空（不放业务文件） | controller / service / spec |
| `service/` | Nest `*.service.ts`、编排型 `*.extractor.ts` | 供应商 HTTP 细节（下沉 provider） |
| `provider/` | `*.provider.ts` | Prisma 编排（除非现有已如此且不扩 scope） |
| `utils/` | `*.util.ts` 等纯函数 | 带路由的 controller |
| `types/` | `*.types.ts`、schemas、prompt、**DTO** | 可执行业务 |
| `processor/` | BullMQ `*.processor.ts` | REST |
| `test/` | 全部 `*.spec.ts` | 生产实现 |

各应用域**不放** controller，避免「有的域有、有的没有」的不对称；也不建 `http/` 目录。

### 2.3 Import 规则

- 禁止两层及以上相对路径 `../../`；域内实现一律 `@/files/ai/<域>/<类型>/文件`；controller 一律 `@/files/ai/<name>.controller`。
- 跨域复用：引用**宿主域**路径，例如 OCR：`@/files/ai/index/provider/vision.provider`。
- 不新增 barrel `index.ts`（除非原先已有且必要）。

## 3. 能力与 DTO 宿主归属

跨域需要同一能力时，**允许**其它应用域 / 其它 Nest 模块 import 宿主路径；**不允许**在 `ai/` 根再开共享层。

| 原位置 / 能力 | 宿主应用域 | 目标子目录 | 说明 |
|---------------|------------|------------|------|
| `asr/*` | **index** | provider / service / utils / types / test | 索引流水线转写 |
| `chunk/*` | **index** | service（extractor、chunker）/ utils（pdf、docx）/ test | 文档解析与切块 |
| `vision/vision.provider`（OCR） | **index** | provider | 图/扫描 PDF OCR；错题本等从 index 引用 |
| `embedding/embedding.provider` + `similarity.util` | **index** | provider + utils | 文本向量；RAG/搜索/助手/知识库从 index 引用 |
| `embedding/image-embedding.provider` | **image-search** | provider | 以图搜图 |
| `vision/math-vision.provider` | **math** | provider | 解题 VL；math-temp / 错题追问从 math 引用 |
| `dto/file-ai-chat.dto.ts` | **chat** | types | 聊天契约；根上 controller / math-temp 从 chat 引用 |
| `utils/chat-model.provider` | **chat** | provider | LLM 入口 |
| `utils/chat-message.util` | **chat** | utils | 消息校验 |
| `utils/structured-object.util` | **chat** | utils | 结构化生成；summary/knowledge 引用 |
| `utils/llm-schema.util` | **chat** | utils | schema 辅助 |
| `utils/env.util` | **chat** | utils | AI 环境变量读取的唯一宿主；其它域 provider 引用此处 |
| `utils/detect-chinese.util` | **translate** | utils | 翻译域 |
| `fingerprint/*` | **image-search** | service | 图像指纹 |

### 3.1 宿主域目标树（示意）

**index（能力最重）**

```
index/
  service/     # files-ai-index、queue、text-extractor、text-chunker、audio-transcript.extractor…
  processor/   # document-index.processor
  provider/    # asr、embedding、vision(OCR)
  utils/       # media-audio、similarity、pdf-*、docx-*
  types/       # queue types、asr.types
  test/        # 全部 spec
```

**chat（LLM + DTO 宿主）**

```
chat/
  service/     # files-ai、rag、chat-session
  provider/    # chat-model
  utils/       # chat-message、structured-object、llm-schema、env
  types/       # file-ai-chat.dto
  test/
```

**tts（样板：无 controller，实现全在类型子目录）**

```
tts/
  provider/tts.provider.ts
  service/files-ai-tts.service.ts
  utils/tts-access.util.ts
  test/
```

**`ai/` 根上的 controller（与域目录平级）**

```
ai/
  files-ai.controller.ts           # DTO from chat/types
  image-search.controller.ts
  files-ai-tts.controller.ts
  files-ai-assistant.controller.ts
  index/ …
  chat/ …
  …
```

## 4. 执行流程（强制）

### 4.1 总顺序

1. **先能力层（C1～C7）**，一次只做一个模块。  
2. **再应用层（A1～A8）**，默认一次一个应用域。  
3. 每一步：搬家 → 改全库 import → jest → **手测说明** → **等用户确认** → 下一步。

### 4.2 单步门禁（不可跳过）

对每一个 C# / A#：

1. 只改当前步骤涉及的文件与引用。  
2. 在 `file_management_backend_nest` 下运行相关 jest（见 §5），**全部通过**。  
3. 用中文向用户给出**手测清单**（入口、操作、预期）。  
4. 明确要求用户回复「通过 / 继续 / 下一个」类确认。  
5. **用户未确认前禁止开始下一步**（含「测试绿了就继续」「顺手把下一个也搬了」等合理化）。

### 4.3 Agent 纪律

- 禁止在同一次回复中完成两个能力模块的搬家。  
- 用户说「执行整个计划」≠ 可以连做多步；仍逐步停等。  
- 失败的测试必须先修到绿，再给手测说明。
