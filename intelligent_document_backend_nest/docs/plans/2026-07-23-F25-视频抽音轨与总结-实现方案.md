# F-25 视频抽音轨 / 总结 / AI 字幕 — 实现方案

> 日期：2026-07-23  
> 状态：实现中（2026-07-24 可演示 MVP：视频准入 + 抽轨 ASR + 播放器文稿/字幕/摘要 Tab）  
> 关联 PRD：[2026-07-08-ai-features-prd.md](./2026-07-08-ai-features-prd.md) § F-25 / §1.5  
> 音频已交付：[2026-07-21-F25-音频转文字-实现方案.md](./2026-07-21-F25-音频转文字-实现方案.md) · [ASR 设计](./2026-07-22-F25-音频转文字-ASR-设计.md)  
> Kickoff：UX **方案 C** + 技术 **路线 Y1**（feature-kickoff 2026-07-23）

**Goal：** 网盘内视频可「建立索引」→ 抽音轨 → ASR 文稿 → 结构化摘要；播放器内文稿/摘要可跳转，并用同源数据做 **WebVTT 跟读字幕** 与 **SRT 导出**。

**Architecture：** 扩展现有 `document-index` 流水线；`prepareAudioForAsr` 已对输入 `.noVideo()`，视频文件可直接进同一转码/ASR 路径。前端增强现有 `VideoPlayerDialog`，不新开播放器。

**Tech Stack：** Nest + BullMQ Worker、fluent-ffmpeg、现有 ASR、现有 Summary Map-Reduce、Vue3 `VideoPlayerDialog`、WebVTT/`<track>`、SRT 文本导出。

---

## 1. 已确认的决策

### 1.1 UX 方案（用户选定）

- **选定：方案 C** — 播放器内「文稿 + 总结」+ AI 字幕（跟读 + 可导出）
- **用户主路径：** 打开视频 → 建立索引（可选体裁）→ 看进度 → Tab「文稿」点句跳转 / Tab「摘要」点章节跳转 → 播放时字幕跟读 → 可下载 `.srt`
- **未选方案（归档）：**
  - A（仅对齐音频文稿）— 缺少总结演示面
  - B（文稿+总结、无字幕）— 用户明确要字幕能力

### 1.2 技术方案（用户选定）

- **选定：路线 Y1** — 扩展现有 `document-index`，不新建独立视频队列
- **关键技术点：** `isIndexableVideo` 准入；Worker 视频走抽音轨+ASR；时长上限；前端 Tab + VTT track + SRT；复用现有摘要流水线
- **未选路线（归档）：**
  - Y2 独立队列 — 工期过长、与音频体验分裂
  - Y3 字幕纯前端、无服务端 format — 可作为实现细节（优先前端拼 VTT/SRT；服务端导出可选）

### 1.3 与 PRD 的对齐说明

- PRD 草案中的 `DocumentSummaryType: video_overview|…`：**MVP 不新增枚举**，复用现有 `chunk/chapter/book` + `summaryGenre`（与音频占位策略一致），避免为视频单独涨迁移面。
- 章节跳转：优先用摘要章节标题 + 对应时间窗（由该章覆盖的 chunk `startMs` 推导）；若推导失败则仅展示摘要、跳转降级为文稿点击。
- 内嵌字幕优先、外链平台、F-41 纠偏：**本期不做**。

## 2. 范围

### 2.1 In Scope

- 网盘已上传常见视频（至少 `mp4` / `webm` / `mov`；扩展名 + MIME 双判）
- `POST .../ai/index` 允许视频；Worker：下载 → ffmpeg 抽音轨/转 16k mono → ASR → 时间戳 chunks → embedding → 现有摘要 Map-Reduce
- 时长/体积上限（env），超限拒绝入队或任务 `failed` + 中文原因
- `VideoPlayerDialog`：文稿 Tab（已有增强）、摘要 Tab、播放时 `<track kind="subtitles">`、导出 SRT
- 单测（准入 / 字幕格式化 / 时长校验）+ e2e mock 视频索引路径 + 手测清单

### 2.2 Out of Scope

- YouTube / B 站链接、yt-dlp
- 解析视频内嵌/外挂软字幕优先于 ASR
- F-41 文稿纠偏与手改
- 独立视频队列、本机 Whisper
- 新增 `video_overview` 等 Prisma 枚举值（可二期）

## 3. 用户操作对照

| 步骤 | 用户动作 | 界面反馈 | 失败/边界 |
|------|----------|----------|-----------|
| 1 | 双击/预览打开视频 | `VideoPlayerDialog` 播放 | RMVB 等继续现有不支持提示 |
| 2 | 点「建立索引」（可选体裁；MVP 可先固定/沿用 novel 或露出选择器） | 按钮 loading；状态文案：准备音轨 / 转写 / 摘要… | 超时长、无音轨、ASR 失败 → 明确错误 |
| 3 | 索引 ready | 文稿列表出现；可点句 seek | ready 但无分句 →「暂无转写分句」 |
| 4 | 切「摘要」Tab | 一句话/章节列表；点章节 seek | 摘要未生成完 → 提示等待 |
| 5 | 播放 | 字幕轨跟读（WebVTT） | 无 segments 则不挂 track |
| 6 | 导出字幕 | 下载 `.srt` | 无文稿则按钮禁用 |

## 4. 技术设计

### 4.1 架构与数据流

```
用户点击「建立索引」
  → FilesAiIndexService：校验 isIndexableVideo（或 audio/文档）
  → 可选：ffprobe 时长 > AI_MAX_VIDEO_DURATION_SEC → 400
  → 入队 document-index
  → DocumentIndexProcessor
       → video：progressMsg「正在抽取音轨」
       → 下载到临时文件
       → prepareAudioForAsr(videoPath)  // 已有 .noVideo() + 16k mono wav
       → ASR → segments → DocumentChunk(startMs/endMs)
       → embedMany → summaryMapReduce（现有）→ ready
  → 前端轮询 status；拉 transcript；拼 VTT/SRT；拉 summary
```

### 4.2 API

| 方法 | 路径 | 请求 | 响应 | 鉴权 |
|------|------|------|------|------|
| POST | `/api/files/:id/ai/index` | `{ summaryGenre, force? }` | 任务状态 DTO | JWT；**扩展允许 video MIME** |
| GET | `/api/files/:id/ai/index/status` | — | 现有 status | JWT |
| GET | `/api/files/:id/ai/transcript` | — | `{ segments: [{ text, startMs, endMs }] }` | JWT（已有） |
| GET | `/api/files/:id/ai/summary` | 现有 query | 现有摘要 DTO | JWT（已有） |
| GET | `/api/files/:id/ai/transcript.srt`（可选） | — | `text/srt` 附件 | JWT；**可二选一：仅前端生成则本行不做** |

鉴权与现有 files AI 路由一致。

### 4.3 数据 / 存储

- **新表：** 无  
- **新字段：** MVP 无强制 Prisma 变更（`startMs`/`endMs` 已有）  
- **复用：** `DocumentIndexJob`、`DocumentChunk`、`DocumentSummary`、现有 `SummaryGenre`  
- **环境变量：**
  ```env
  AI_MAX_VIDEO_DURATION_SEC=1800   # MVP 建议 30 分钟；可配置
  AI_MAX_VIDEO_FILE_BYTES=         # 可选，与上传上限对齐时可省略
  ```

### 4.4 前端

- **入口：** `VideoPlayerDialog`（`file_management_frontend/src/components/VideoPlayerDialog/index.vue`）  
- **修改/新增：**
  - 同文件：摘要 Tab；体裁选择（若后端要 genre）；字幕 `<track>`；导出按钮  
  - 工具：`src/utils/subtitleFormat.ts`（或 `components/VideoPlayerDialog/subtitleFormat.ts`）— `segments → VTT/SRT`  
  - API：沿用 `triggerDocumentIndex` / `getFileTranscript` / `get…Summary`（`src/api/ai.ts`）  
  - i18n：`preview.*` 下补视频索引/字幕相关文案  
- **状态：** 复用现有 index 轮询；`mediaKind === 'video'` 时启用摘要 Tab + track

### 4.5 后端

| 区域 | 路径 | 动作 |
|------|------|------|
| 准入 | `src/files/ai/index/service/text-extractor.ts` | 新增 `isIndexableVideo`（MIME/扩展名） |
| 入队校验 | `src/files/ai/index/service/files-ai-index.service.ts` | 允许 video；时长探测失败/超限 → `BadRequestException` |
| 抽轨 | `src/files/ai/index/utils/media-audio.util.ts` | 确认视频输入可用；必要时补 `ffprobeDurationSec` |
| 抽取编排 | `src/files/ai/index/service/audio-transcript.extractor.ts` 或新建 `media-transcript.extractor.ts` | 视频与音频共用「落盘 → prepare → ASR」；入口按 MIME 分支文案 |
| Worker | `src/files/ai/index/processor/document-index.processor.ts` | `isIndexableAudio \|\| isIndexableVideo` 走转写分支；progressMsg 区分抽轨/转写 |
| Controller | `src/files/ai/files-ai.controller.ts` | 错误文案；可选 SRT 路由 |
| 模块注册 | `files.module.ts` / worker module | 通常无需新 provider |

### 4.6 错误与边界

| 场景 | 行为 |
|------|------|
| 不支持的视频扩展名 | 入队 400 |
| 时长超限 | 入队 400 或任务 failed（优先入队前拒绝） |
| 无音轨 / 静音导致空转写 | failed：「未识别到可用语音」 |
| ASR/ffmpeg 失败 | failed + 截断后的上游信息 |
| 浏览器无法硬解视频 | 保持现有提示；索引仍可后台完成 |
| 二次索引同 hash | 现有 Conflict 逻辑 |

## 7. 测试计划

- **单测：** `isIndexableVideo`；时长超限；`segmentsToVtt` / `segmentsToSrt` 时间轴边界；processor 视频分支 mock extractor  
- **e2e：** mock 队列 + 视频 MIME 触发 index；transcript 非空（可与 `files-ai-index.e2e-spec` 扩展）  
- **手测：** 真实短 mp4（有对白）→ 索引 → 文稿/摘要/字幕/导出；无对白视频看失败文案；超长文件（或临时调低 env）看拒绝

## 8. 风险与后续

| 风险 | 缓解 |
|------|------|
| 长视频贵且慢 | 严格时长上限；progressMsg 分段提示 |
| ASR 错字在字幕上更刺眼 | 接受 MVP 质量；F-41 纠偏不挡 ⚑ |
| 摘要章节无精确 startMs | 用 chunk 时间窗推导；推不到则只展示 |
| ffmpeg 抽轨失败（编码怪） | failed 文案 + 建议转码为 mp4 |

**明确延后：** 内嵌字幕优先、外链、F-41、`video_*` SummaryType 枚举、服务端必选 SRT 路由（前端导出足够时可砍）。

**⚑ 关系：** 本方案交付后，§1.5「视频」段可勾选；其后才是 F-24 MCP。
