## 3. 文件分享（链接分享）

**本节当前无未完成项。**

**产品定界（2026-04-19）**：原 §3(3)「仅查看 / 可下载 / 可编辑」**已从需求中移除**。理由：网盘**无**通用在线编辑，「仅查看、不可下载」对产品价值有限；链接分享统一按 **可下载** 能力验收，与当前实现一致。详见 [REQUIREMENTS.md](./REQUIREMENTS.md) §3（3）。

**说明**：§3(1) 公开链接 / 私密（提取码）、分享页与 §3(2) 有效期（含过期校验与定时清理过期分享），以及 §3（3）所定义的 **可下载向链接分享**，**已在当前版本实现**。**§3(4)**：访客行为写入 `ShareAccessLog`；分享者通过顶栏 **「我的分享」**（`GET /api/shares/mine`、`GET /api/shares/:id/access-logs`）查看列表与访问记录。好友会话内传文件与「链接分享」不是同一模块。

**访问人数上限 `max_visitors`（已实现，不列入上表）**：创建分享时可设 1～10 或不限；服务端在 **查看列表 / 下载 / 转存** 路径用事务 + `file_shares` 行锁校验——**未登录**访问仅按 **IP**（`visitor_id` 为 NULL 的日志）；**转存须登录**，仅按 **用户 ID**（`visitor_id` 非空），与未登录访问**分开计数**（无匿名转存）。实现见 `share.controller.ts` 中 `ensureVisitorAllowedInTx`；`share_access_logs.action` 含 `view` / `download` / `save`。
