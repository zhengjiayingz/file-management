本项目是仿百度云的文件管理系统。主要功能支持文件的上传和下载。

## 功能需求

### 1. 用户管理
（1）用户分成普通用户，VIP用户和管理员用户。支持用户的注册升级vip用户，普通用户的存储配额是1GB，VIP用户的存储配额是2GB（由于是在个人电脑上开发，存储空间有限，暂时配这么多）

（2）已使用配额以进度条百分比的形式在用户界面中展示。

（3）多用户之间支持添加好友，好友之间支持非实时的收发消息。可以给好友分享文件。

（4）好友申请需要对方审核同意后才能成为好友。

（5）消息支持已读状态显示。

（6）新消息提醒：支持浏览器通知提醒新消息。

（7）用户注册强制密码强度，必须符合8位以上密码。包含数字，字母，大小写，特殊字符4个要求中至少满足3个

（8）密码以SHA256形式存储在数据库。支持重置密码（重置的时候重置为同一的初始密码），修改密码（修改密码的时候需要输入原密码）

（9）采用双token无感登录机制：
   - Access Token：短期有效（15分钟），用于API请求认证
   - Refresh Token：长期有效（7天），用于刷新Access Token
   - Access Token过期时，前端自动使用Refresh Token获取新的Access Token，用户无感知
   - Refresh Token过期或被撤销时，需要重新登录
   - 支持主动登出，登出时清除所有token

### 2. 文件管理

（1）需要支持的文件类型：各种办公软件文档，如.doc,.docx,.ppt,.pptx,.xls,.xlsx,.pdf,txt，常见的视频、音频文件、图片文件，以上类型需要支持在线预览（具体范围以实现为准）。图片文件上传的时候，支持裁剪后上传。**压缩包内单个条目不提供「像普通文件一样」的在线预览**；**VIP/管理员**进行在线解压时，可在弹窗中查看**包内文件/文件夹列表**（仅用于勾选解压，非通用预览器）。压缩包的上传、下载及在线解压规则见**（1-1）**。

（1-1）**压缩包与在线解压（VIP 与管理员权益，解压到当前网盘目录）**
   - 系统支持压缩包文件的**上传与普通下载**（与普通文件一致）。
   - **在线解压**：将 ZIP 内**用户勾选的部分文件**解压到用户**当前浏览的网盘目录**（根目录或某一文件夹），**不**再提供「整包解压后打包成 ZIP/文件夹下载」的能力（与「下载不支持文件夹」等产品约束一致）。
   - **权限**：仅 **VIP 用户**与 **管理员**可使用在线解压；**普通用户**双击压缩包**不**打开解压界面，**直接下载**原始压缩包。
   - **VIP/管理员**双击**可在线解压的压缩包**时：打开**解压对话框**。
     1. 前端请求后端列出包内条目（树形展示，可勾选文件或文件夹；勾选文件夹等价于其下待解压的文件集合）。
     2. 用户点击「解压选中项」后，前端先请求后端做**重名预检**（仅根据当前网盘目录结构判断，**不解压、不读取压缩包实体内容**）：
        - **无重名**：直接执行解压，将选中文件写入当前网盘目录（保持包内相对目录结构所需的文件夹会按需创建）。
        - **有重名**：弹出二次确认，列出可能冲突的包内路径，用户选择处理方式（与上传同名策略对齐）：
          - **自动重命名**：同名时改为 `名称(1).ext` 等形式；
          - **保存为新版本**：覆盖已有同名文件并写入版本历史；
          - **保留两者**：网盘中存在两个**同名**文件记录（与分片合并/秒传在「保留两者」下的行为一致）。
   - **格式**：后端在线解压**仅支持 ZIP 容器**（扩展名常见为 `.zip`，以及 `.jar`/`.war`/`.ear` 等；亦参考存储的 MIME）。**非 ZIP**（如 RAR、7z）：VIP/管理员若尝试在线解压，前端提示并改为**直接下载原包**；普通用户始终直接下载。
   - **限制（以实现为准）**：单包体积上限、单次解压文件数上限、列表条目数上限等由后端配置；超大或加密 ZIP 可能无法列出或解压。

（2）由于支持多用户，所以多个用户上传同一个文件（以文件指纹进行识别），在服务器中只保存一份原始文件。多个用户共享同一个文件的时候，以数据库记录引用的形式指向该文件。

（3）文件上传的时候需要进行分片上传。

（4）支持断点续传，秒传功能。

（5）上传下载需要有用户历史记录列表

（6）当一个文件中再无用户关联该文件的时候，物理文件在服务器上保留24小时。如果过了24小时还是没有用户引用的状态，则物理删除该文件。

（7）支持文件夹管理，包括创建文件夹、移动文件、重命名、文件夹层级结构。

（8）支持文件搜索功能，可按文件名、类型、上传时间进行搜索。

（9）支持文件排序，可按名称、大小、时间、类型排序。

（10）支持批量操作，包括批量下载、删除、移动文件。（**实现要点**：批量下载为服务端将选中项打包为 ZIP，支持选中文件夹并递归子文件；批量删除、移动为后端批量接口，在事务内一次或批量更新数据库，详见上文「实现状态总览」。）

（11）回收站功能：逻辑删除的文件进入回收站，支持恢复和彻底删除。
   - 还原机制：还原文件时，优先还原到原文件夹。
   - 智能归位：若原文件夹已不存在或也在回收站中，系统自动将文件还原到根目录，并给出相应提示。

（12）文件版本管理：同名文件上传时保留历史版本（可选功能）。

（13）文件标签/分类：支持给文件打标签便于管理。

（14）拖拽上传：支持拖拽文件到页面进行上传。

（15）上传队列管理：显示上传进度、支持暂停/继续/取消上传。

（16）缩略图：图片和视频文件显示缩略图。

（16-1）**网格视图视频预览**：在网格展示文件时，对视频文件采用**鼠标悬停**在该文件项（卡片）上时，**静音循环播放前 10 秒**作为动态预览；未悬停时仅显示静态缩略图（poster）。**设计说明**：采用悬停触发而非进入视口自动播放，是为了**控制网络与解码流量**，避免大量视频同时拉流。

（17）文件预览优化：大文件分页预览、视频播放进度记忆。

### 3. 文件分享

（1）支持生成分享链接，包括公开链接和私密链接（需要提取码）。

（2）分享链接支持设置有效期（1天、7天、30天、永久）。

（3）分享权限控制：仅查看、可下载、可编辑。

（4）分享记录：可查看谁访问了分享链接，访问时间等信息。

### 4. 权限管理

（1）普通用户和VIP用户都只支持管理自己的文件。当用户删除自己文件的时候，只在数据库记录中进行逻辑删除。

（2）管理员用户存储空间不受限制。管理员除了可以管理自己的文件之外，还支持手动清理没有人引用的服务器文件。

### 5. 安全性

（1）双token认证机制：Access Token + Refresh Token，提供安全的无感登录体验。

（2）Token管理：支持token刷新、撤销、黑名单机制。

（3）登录日志：记录用户登录时间、IP地址、设备信息。

（4）异常登录提醒：检测异地登录并提醒用户。

（5）文件加密：敏感文件支持加密存储（可选功能）。

（6）操作日志：记录重要操作，包括上传、删除、分享等。

（7）会话管理：支持查看和管理所有活跃会话，可远程登出指定设备。

### 6. 管理员功能

（1）用户管理：查看所有用户列表、禁用/启用用户账号。

（2）存储统计：查看系统总存储使用情况、用户存储使用排行。

（3）文件审核：违规文件检测和处理功能。

## 业务流程

### 文件上传流程
1. 前端计算文件MD5（文件指纹）
2. 请求后端检查文件是否已存在（秒传检测）
3. 如果文件已存在，直接创建用户文件引用记录
4. 如果文件不存在，开始分片上传
5. 支持断点续传，记录已上传的分片
6. 所有分片上传完成后，后端合并分片
7. 创建物理文件记录和用户文件引用记录

### 文件删除流程
1. 用户删除文件，进行逻辑删除（移入回收站）
2. 回收站文件可以恢复或彻底删除
3. 彻底删除时，删除用户文件引用记录
4. 检查物理文件是否还有其他用户引用
5. 如果没有引用，标记物理文件为待删除状态
6. 定时任务检查待删除文件，超过24小时则物理删除

### 文件分享流程
1. 用户选择文件生成分享链接
2. 设置分享权限和有效期
3. 生成唯一分享码
4. 其他用户通过链接访问
5. 记录访问日志
6. 根据权限控制可执行的操作

### 在线解压到网盘流程（VIP/管理员，ZIP）
1. 用户在网盘中双击 ZIP，打开解压对话框；前端请求 `GET /api/files/:id/archive/entries` 展示包内条目树。
2. 用户勾选要解压的文件（或文件夹对应的子文件），点击「解压选中项」。
3. 前端请求 `POST /api/files/:id/archive/conflicts`，提交 `parentId`（当前目录）与 `paths`（包内文件路径列表）；返回是否重名及冲突路径列表。
4. **无重名**：`POST /api/files/:id/archive/extract`，`conflictAction` 可为默认策略（如 `suffix`），后端将临时解压后的文件注册到网盘（`registerLocalFileInDrive` 等）。
5. **有重名**：用户二次选择 `version` / `duplicate` / `suffix` 后，再调用 `archive/extract` 完成解压与落库。
6. 后端使用 **yauzl** 读取 ZIP 目录与流式解压；操作记入操作日志（以实现为准）。

### 文件版本管理流程
1. 用户上传同名文件（前端通过 `check-name` 检测当前目录下是否已有同名**文件**）
2. 前端弹出冲突处理对话框（与 Element Plus 交互一致，关闭对话框视为放弃本次上传）：
   - **保存为新版本**：`conflictAction = version`，后端将当前文件归档到 `file_histories`，更新 `user_files` 指向新存储并 `version + 1`
   - **保留两者**：`conflictAction = rename`（命名沿用上传接口字段），后端**新建**第二条 `user_files` 记录，**文件名可与已有文件相同**（同名并存）
3. 版本更新逻辑（选择「保存为新版本」时）：
   - 归档当前版本信息到 `file_histories` 表
   - 将新文件信息更新到 `user_files` 表
   - `version` 号自动 +1
4. 查看历史版本：
   - 展示版本列表（包含：版本号、归档时间、文件大小、文件名）
   - **在线预览**：双击历史版本记录即可在新标签页预览文件内容
   - **下载**：支持下载指定的历史版本文件
   - **回滚**：支持回滚到旧版本（回滚操作本身也会生成新的版本记录，保持线性历史）

## 技术要求

前端技术：vue3 + TypeScript + Vite + ElementPlus

## 其他需求

（1）支持主题切换，包括浅色模式和深色模式。用户的个性化主题设置需持久化到数据库，支持跨设备同步。
 
 （2）支持国际化，包括简体中文，繁体中文，英文。用户的语言偏好需持久化到数据库，登录后自动应用。
