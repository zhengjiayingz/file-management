# 文件管理系统需求文档

## 项目概述
本项目是仿百度云的文件管理系统。主要功能支持文件的上传和下载。

## 功能需求

### 1. 用户管理
（1）用户分成普通用户、VIP 用户和管理员。普通用户可**提交 VIP 升级申请**，经**管理员审核**通过后成为 VIP（非注册环节直接升级）。**新注册普通用户**默认配额、**审核通过后 VIP 默认配额**以数据库 **`system_settings`** 中 `storage_quota_user_bytes`、`storage_quota_vip_bytes` 为准（管理员在 **管理员看板 → 系统管理** 中配置；Prisma 默认值约为 **1GB / 2GB**，可改）。

（2）已使用配额以进度条百分比的形式在用户界面中展示。

（3）多用户之间支持添加好友，好友之间支持收发消息（**当前实现**：基于 WebSocket 的 **Socket.IO 实时推送**，含新消息、好友关系变更同步；会话展示与已读、未读汇总联动。历史需求曾表述「非实时」，以现网实现为准）。可以给好友分享文件。

（4）好友申请需要对方审核同意后才能成为好友。

（5）消息支持已读状态显示。

（6）用户注册强制密码强度：**最短长度**（默认 8 位以上，可配置）、在管理员勾选的 **数字 / 小写 / 大写 / 特殊字符** 类别中，密码须**至少满足其中 M 类**（M 与勾选集合均由 **`system_settings`** 配置；默认场景可等价于历史上「四类中选三类」）

（7）密码以 SHA256 形式存储在数据库。支持**管理员重置**为统一临时密码（用户登录后须强制改为符合策略的新密码，首次改密可不填当前密码）；支持 **`change-password` 接口**在已登录状态下修改密码（非强制改密场景须校验原密码）。**设置弹窗**提供自助修改密码：须正确输入原密码、新密码与确认，新密码须符合与注册一致的强度规则。

（8）采用双token无感登录机制：
   - Access Token：短期有效（15分钟），用于API请求认证
   - Refresh Token：长期有效（7天），用于刷新Access Token
   - Access Token过期时，前端自动使用Refresh Token获取新的Access Token，用户无感知
   - Refresh Token过期或被撤销时，需要重新登录
   - 支持主动登出，登出时清除所有token
   - **（实现）并发会话与踢设备**：每条 `refresh_tokens` 记录对应一个浏览器/设备会话。**普通用户**同时最多 **2** 个活跃会话，**VIP** 最多 **5** 个，**管理员**不限制。超过上限时新设备登录返回 **409**（`code: SESSION_LIMIT`），前端弹窗展示当前活跃会话列表，用户可选择 **踢掉其中一条** 后在本机登录（请求体带 `revokeSessionId`）；踢除时对该行 `is_revoked = true` 且 **`users.session_version` 递增**，被踢端旧 JWT 与 Refresh 均失效。弹窗打开时，密码框回车/主「登录」按钮走踢人登录，避免重复发起未带 `revokeSessionId` 的登录。**已登录后**：顶栏用户菜单 **「会话管理」** 可随时列出活跃会话，多选后 **登出** 所选设备（`POST /api/auth/sessions/list`、`POST /api/auth/sessions/revoke`；登出为逻辑撤销，非物理删行）。

### 2. 文件管理

（1）需要支持的文件类型：各种办公软件文档，如.doc,.docx,.ppt,.pptx,.xls,.xlsx,.pdf,txt，常见的视频、音频文件、图片文件，以上类型需要支持在线预览（具体范围以实现为准）。图片文件上传的时候，支持裁剪后上传。**压缩包内单个条目不提供「像普通文件一样」的在线预览**；**VIP/管理员**进行在线解压时，可在弹窗中查看**包内文件/文件夹列表**（仅用于勾选解压，非通用预览器）。压缩包的上传、下载及在线解压规则见**（1-1）**。

（1-1）**压缩包与在线解压（VIP 与管理员权益，解压到当前网盘目录）**
   - 系统支持压缩包文件的**上传与普通下载**（与普通文件一致）。
   - **在线解压**：将 ZIP 内**用户勾选的部分文件**解压到用户**当前浏览的网盘目录**（根目录或某一文件夹），**不**再提供「整包解压后打包成 ZIP/文件夹下载」的能力（与「下载不支持文件夹」等产品约束一致）。
   - **权限**：仅 **VIP 用户**与 **管理员**可使用在线解压；**普通用户**双击压缩包**不**打开解压界面，**直接下载**原始压缩包。
   - **VIP/管理员**双击**可在线解压的压缩包**时：打开**解压对话框**。
     1. 前端请求后端列出包内条目（树形展示，可勾选文件或文件夹；勾选文件夹等价于其下待解压的文件集合）。
     2. 用户点击「解压选中项」后，前端先请求后端做**重名预检**（仅根据当前网盘目录结构判断，**不解压、不读取压缩包实体内容**）：
        - **无重名**：直接执行解压，将选中文件写入当前网盘目录（保持包内相对目录结构所需的文件夹会按需创建）。
        - **有重名**：弹出二次确认，列出可能冲突的包内路径，用户选择处理方式（与上传同名策略对齐）：
          - **自动重命名**：同名时改为 `名称(1).ext` 等形式；
          - **保存为新版本**：覆盖已有同名文件并写入版本历史；
          - **保留两者**：网盘中存在两个**同名**文件记录（与分片合并/秒传在「保留两者」下的行为一致）。
   - **格式**：后端在线解压**仅支持 ZIP 容器**（扩展名常见为 `.zip`，以及 `.jar`/`.war`/`.ear` 等；亦参考存储的 MIME）。**非 ZIP**（如 RAR、7z）：VIP/管理员若尝试在线解压，前端提示并改为**直接下载原包**；普通用户始终直接下载。
   - **限制（以实现为准）**：单包体积上限、单次解压文件数上限、列表条目数上限等由后端配置；超大或加密 ZIP 可能无法列出或解压。

（2）由于支持多用户，所以多个用户上传同一个文件（以文件指纹进行识别），在服务器中只保存一份原始文件。多个用户共享同一个文件的时候，以数据库记录引用的形式指向该文件。

（3）文件上传的时候需要进行分片上传。

（4）支持断点续传，秒传功能。

（5）上传下载需要有用户历史记录列表

（6）当一个文件中再无用户关联该文件的时候，物理文件在服务器上保留24小时。如果过了24小时还是没有用户引用的状态，则物理删除该文件。

（7）支持文件夹管理，包括创建文件夹、移动文件、重命名、文件夹层级结构。

（8）支持文件搜索功能，可按文件名、类型、上传时间进行搜索。

（9）支持文件排序，可按名称、大小、时间、类型排序。

（10）支持批量操作，包括批量下载、删除、移动文件。（**实现要点**：批量下载为服务端将选中项打包为 ZIP，支持选中文件夹并递归子文件；批量删除、移动为后端批量接口，在事务内一次或批量更新数据库，详见上文「实现状态总览」。）

（11）回收站功能：逻辑删除的文件进入回收站，支持恢复和彻底删除。
   - 还原机制：还原文件时，优先还原到原文件夹。
   - 智能归位：若原文件夹已不存在或也在回收站中，系统自动将文件还原到根目录，并给出相应提示。

（12）文件版本管理：同名文件上传时保留历史版本（可选功能）。

（13）文件标签/分类：支持给文件打标签便于管理。**普通用户 / 有效 VIP** 可创建的标签总数受 **`system_settings.max_tags_user`** / **`max_tags_vip`** 约束（管理员不限制）。

（14）拖拽上传：支持拖拽文件到页面进行上传。

（15）上传队列管理：显示上传进度、支持暂停/继续/取消上传。

（16）缩略图：图片和视频文件显示缩略图。

（16-1）**网格视图视频预览**：在网格展示文件时，对视频文件采用**鼠标悬停**在该文件项（卡片）上时，**静音循环播放前 10 秒**作为动态预览；未悬停时仅显示静态缩略图（poster）。**设计说明**：采用悬停触发而非进入视口自动播放，是为了**控制网络与解码流量**，避免大量视频同时拉流。

（17）文件预览优化（**以当前实现为准，2026-04-24 起与进度报告一致**）：
   - **音频、视频播放进度记忆**：已由其它条目验收，不在此条重复要求。
   - **Word / PPT / ODP**：服务端 **LibreOffice** 无头转 **PDF** 后内嵌浏览器预览；PPT/ODP 为降低首屏等待，**先**导出前 **N** 页为快览（默认约 25 页，可环境变量 `PPT_PREVIEW_FIRST_SLIDES`），**后台队列**再转**全文** PDF；`GET …/preview-state` 供前端轮询 `partial → full` 后**刷新** iframe；全文转 PDF 超时**单独**放宽（`PPT_FULL_CONVERSION_MAX_MS` 等），避免长期停在快览页。
   - **PDF**：在允许的类型内**直接**流式预览；缓存策略与 Office 快览/全文**区分**（快览不长期强缓存，便于接全文）。
   - **TXT/MD 等白名单文本**：`GET /api/files/:id/text-chunk` 按**字节分块**读取，在 **UTF-8 码点边界**截断，避免块衔接处**乱码**；配合前端分段展示，按产品视为满足「大文本」**分段预览**能力。
   - **说明**：**Excel**（.xls/.xlsx）产品范围内以**本地下载**为主，不强制走与 Word/PPT 同一套内嵌表预览。若以后单独要求 **GB 级单文件**纯文本的「虚拟页码/无限滚动」式极致体验，可另立增强项，**不**与本条当前验收冲突。

### 3. 文件分享

（1）支持生成分享链接，包括公开链接和私密链接（需要提取码）。

（2）分享链接支持设置有效期（1天、7天、30天、永久）。

（3）**链接分享能力范围**：访客通过分享链接可浏览分享内容并**下载**（及转存等已实现能力）。**不**要求「仅查看、不可下载」与「访客可编辑」等分档；全站**不**将「分享文件在线编辑」纳入产品范围（网盘内亦无通用在线编辑能力）。

（4）分享记录：可查看谁访问了分享链接，访问时间等信息。

（5）**访问人数上限**：创建分享时可限制最多访问人数（如 1～10）或不限制（`max_visitors`）。**未登录**打开列表、下载按 **IP** 计数；**转存到个人网盘须登录**，按 **用户 ID** 计数（与未登录访问分开统计，无匿名转存）。实现见 `share.controller.ts` `ensureVisitorAllowedInTx`。

### 4. 权限管理

（1）普通用户和VIP用户都只支持管理自己的文件。当用户删除自己文件的时候，只在数据库记录中进行逻辑删除。

（2）管理员用户存储空间不受限制。管理员管理自己的文件；**无人引用的服务器物理文件**由系统在 **§2(6)** 约定的定时机制（如无引用满 24 小时后物理删除）**自动回收**，**不**要求提供管理员手动清理功能（降低误操作风险与维护成本）。

### 5. 安全性

（1）双token认证机制：Access Token + Refresh Token，提供安全的无感登录体验。

（2）Token管理：支持 token 刷新、撤销、**强失效机制**（**实现**：Refresh 撤销 + Access 与 **`session_version`/`sv`** 绑定；与常见表述中的「黑名单」等效于「可使已签发 Access 立即作废」，**不**要求独立存储逐枚 Access 黑名单）。

（3）登录日志：记录用户登录时间、IP地址、设备信息。

（4）异常登录提醒：检测异地登录并提醒用户。

（5）文件加密：敏感文件支持加密存储（可选功能）。

（6）操作日志：记录重要操作，包括上传、删除、分享等。

（7）会话管理：支持查看和管理所有活跃会话，可远程登出指定设备。**（已实现）** **登录时**会话数达上限：见 **§1(8)** 弹窗踢一条后在本机登录。**已登录后**：顶栏 **「会话管理」** 打开对话框，展示未撤销且未过期的 `refresh_tokens`、多选后批量登出所选设备（调用 `sessions/list`、`sessions/revoke`，并 `session_version` 递增以使被登出端立即失效）。登出与撤销均为 **`is_revoked = true`**，表记录保留不作物理删除（审计/排查）。

（8）**管理员账户可选 TOTP 两步验证（2FA）**：仅 **`role = admin`** 可在 **「设置」→ 登录密码** 中开启；使用标准 **TOTP**（Microsoft Authenticator 等）扫码绑定。开启后**登录**在密码验证通过后不直接发双 Token，而返回 `code: MFA_REQUIRED` 与短时 **`mfaToken`**（JWT），再 **`POST /api/auth/mfa/verify`** 提交 6 位动态码；校验通过后与常规登录一样签发双 Token 并写成功登录日志。会话达上限时，可在第二步同一请求中携带 **`revokeSessionId`**。关闭 2FA 须验证**登录密码 + 当前动态码**，并 **递增 `session_version`** 使各端重登。普通用户与 VIP **不要求、不提供** 全站 2FA（与 [EXTRA_REQUIREMENTS.md](./EXTRA_REQUIREMENTS.md) 第 1 条「全量用户 2FA」为不同范围）。数据库字段与接口见 [DATABASE_DESIGN.md](./DATABASE_DESIGN.md) §3.1、[API_DOCUMENTATION.md](./API_DOCUMENTATION.md)。

### 6. 管理员功能

（1）用户管理：查看所有用户列表、禁用/启用用户账号。

（2）存储统计：查看系统总存储使用情况、用户存储使用排行。

（3）**系统管理（全局配置）**：管理员在 **管理员看板** 的 **「系统管理」** 中维护数据库表 **`system_settings`**（单行 `id=1`）。保存后立即影响后续业务逻辑（**不改变**已成功登录会话的 Access Token；**已存在用户**的当前 `storage_quota` 仅在 **新用户注册**、**VIP 审核通过** 等事件中按此处配置写入；密码策略变更后，注册/改密/登录侧校验与 **`GET /api/auth/password-policy`** 对外摘要一致，与 [DATABASE_DESIGN.md](./DATABASE_DESIGN.md) §3.17、[BUSINESS_FLOWS.md](./BUSINESS_FLOWS.md) §7.4 一致）。配置项包括：

- **密码策略**：最短长度（`password_min_length`）；须包含的字符类别多选（`password_required_categories`，JSON 数组，取值如 `digit` / `lower` / `upper` / `special`）；在**已勾选**类别中至少满足几类（`password_min_categories_in_pool`，可小于勾选数）。
- **各角色默认存储空间（字节）**：普通用户（`storage_quota_user_bytes`）、VIP（`storage_quota_vip_bytes`）、管理员档默认（`storage_quota_admin_bytes`，用于会员中心对比展示等）；管理端界面通常以 GB 编辑并换算为字节持久化。
- **标签数量上限**：普通用户（`max_tags_user`）、VIP 用户（`max_tags_vip`）；**管理员**创建标签不受上述上限约束。

**接口**（均需管理员 JWT）：`GET /api/admin/system-settings`、`PUT /api/admin/system-settings`。会员中心对比表与配置同步展示：`GET /api/vip/tier-config`（需登录）。

（4）文件审核：违规文件检测和处理功能。

## 业务流程

### 文件上传流程
1. 前端计算文件MD5（文件指纹）
2. 请求后端检查文件是否已存在（秒传检测）
3. 如果文件已存在，直接创建用户文件引用记录
4. 如果文件不存在，开始分片上传
5. 支持断点续传，记录已上传的分片
6. 所有分片上传完成后，后端合并分片
7. 创建物理文件记录和用户文件引用记录

### 文件删除流程
1. 用户删除文件，进行逻辑删除（移入回收站）
2. 回收站文件可以恢复或彻底删除
3. 彻底删除时，删除用户文件引用记录
4. 检查物理文件是否还有其他用户引用
5. 如果没有引用，标记物理文件为待删除状态
6. 定时任务检查待删除文件，超过24小时则物理删除

### VIP 升级与审核流程
1. 普通用户发起 VIP 升级申请，系统创建待审核记录（`pending`）。
2. 系统向所有管理员发送站内消息，正文含固定前缀 **`[VIP升级申请]`**，便于会话内识别。
3. **管理员**可在 **通讯录 → VIP申请** 中查看并同意/拒绝；也可在 **与申请人的私聊会话** 中，对该条申请消息使用 **同意 / 拒绝**（与列表审核等价）。
4. **同意**：申请人角色更新为 VIP，**`users.storage_quota`** 更新为当前 **`system_settings.storage_quota_vip_bytes`**（与管理员「系统管理」配置一致）。**拒绝**：该次申请结束，用户可再次申请（以实现为准）。
5. VIP 用户可使用 **在线解压 ZIP** 等与角色绑定的能力（见 §2（1-1））。

### 文件分享流程
1. 用户选择文件生成分享链接
2. 设置有效期（及提取码、`max_visitors` 等，以实现为准）
3. 生成唯一分享码
4. 其他用户通过链接访问
5. 记录访问日志（`view` / `download` / 登录转存 `save`）
6. 访客在链接分享场景下按**可下载**等产品约定执行列表、预览、下载、转存等（无「仅查看 / 可编辑」分档）；**达到访问人数上限**时，后续访问在服务端校验后拒绝

### 在线解压到网盘流程（VIP/管理员，ZIP）
1. 用户在网盘中双击 ZIP，打开解压对话框；前端请求 `GET /api/files/:id/archive/entries` 展示包内条目树。
2. 用户勾选要解压的文件（或文件夹对应的子文件），点击「解压选中项」。
3. 前端请求 `POST /api/files/:id/archive/conflicts`，提交 `parentId`（当前目录）与 `paths`（包内文件路径列表）；返回是否重名及冲突路径列表。
4. **无重名**：`POST /api/files/:id/archive/extract`，`conflictAction` 可为默认策略（如 `suffix`），后端将临时解压后的文件注册到网盘（`registerLocalFileInDrive` 等）。
5. **有重名**：用户二次选择 `version` / `duplicate` / `suffix` 后，再调用 `archive/extract` 完成解压与落库。
6. 后端使用 **yauzl** 读取 ZIP 目录与流式解压；操作记入操作日志（以实现为准）。

### 文件版本管理流程
1. 用户上传同名文件（前端通过 `check-name` 检测当前目录下是否已有同名**文件**）
2. 前端弹出冲突处理对话框（与 Element Plus 交互一致，关闭对话框视为放弃本次上传）：
   - **保存为新版本**：`conflictAction = version`，后端将当前文件归档到 `file_histories`，更新 `user_files` 指向新存储并 `version + 1`
   - **保留两者**：`conflictAction = rename`（命名沿用上传接口字段），后端**新建**第二条 `user_files` 记录，**文件名可与已有文件相同**（同名并存）
3. 版本更新逻辑（选择「保存为新版本」时）：
   - 归档当前版本信息到 `file_histories` 表
   - 将新文件信息更新到 `user_files` 表
   - `version` 号自动 +1
4. 查看历史版本：
   - 展示版本列表（包含：版本号、归档时间、文件大小、文件名）
   - **在线预览**：双击历史版本记录即可在新标签页预览文件内容
   - **下载**：支持下载指定的历史版本文件
   - **回滚**：支持回滚到旧版本（回滚操作本身也会生成新的版本记录，保持线性历史）

## 其他需求

（4）**设置（弹窗）**：通过顶栏用户下拉菜单「设置」打开。**头像**：支持选择图片上传并更新头像；服务端将文件保存至静态资源目录（如 `uploads/avatars/`），用户表字段 `avatar_url` 记录相对访问路径，通过 `POST /api/user/avatar`（multipart，字段名 `avatar`）更新；格式、大小等限制以实现为准。**邮箱**：默认在输入框中只读展示；用户点击「修改」后输入框可编辑，并显示「保存」「取消」；点击「保存」前须进行邮箱格式校验，提交 `PUT /api/user/profile`（`email` 可为空字符串表示清空）；若与已有账号邮箱冲突，接口返回明确错误。**修改密码**：须填写当前密码、新密码与确认；新密码须符合系统密码强度规则；提交 `POST /api/auth/change-password`，成功后刷新 Access Token。**管理员**：在「登录密码」区块可**开启 / 关闭 TOTP 两步验证**（与 §5(8) 一致）。登录后顶栏用户名旁可展示头像（有则显示图片 URL）。
